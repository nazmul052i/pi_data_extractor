DCPCTGOOD

This parameter is used to determine the minimum number of values with good status needed in a dataset for the average of the data to have a good status. The averaging calculations which use this parameter include the running average (DCAVGOUT) calculation by the Process Data Collection (DC) module and the averaging of UPR, BPR, AZUVAZ and SSTOTAL values by the Lab Update (LBU) module.

Applies to:  

Configuration parameters

Data Type:  

Float

Access:  

Read Only

Range:  

0.0 to 100.0

Default:  

75.0 (Required Entry)

Keywords:  

LOCAL CONSTANT

DC History File

The data collection history file stores binary data for a list of points specified using the DC point value (DCPNTVAL) entries. A new set of values is written during each cycle at an update frequency specified in the data collection update frequency (DCUPDFREQ) parameter. In addition to the DCPNTVAL entries, the following values are also added to the data collection history list when the Prediction (PR) and Steady-State Detection (SSD) modules are:

UPR Unbiased Prediction
BPR Biased Prediction
UAZ Unbiased Dynamic Prediction
BAZ Biased Dynamic Prediction
SS Total Percentage Steady State
ISS Individual Percentage Steady State

The maximum number of lab records stored in the DC history file is specified using the DCMAXREC parameter. The DC history backup option, DCBCKOPT, can be configured to automatically generate backup copies of historical data. The current file information can be copied to the backup file several times each day as specified by the backup frequency in hours parameter, DCBCKHRS. The DCBCKVERS parameter specifies the number of daily backup versions to keep.

AZUVLDNUM

Validation message number for the last successful analyzer update. Typically this message will be one of the "Good: " messages listed below. It will change to "Bad: Signal Too Old" message if the time of analyzer update (AZUTMSTR) is older than AVMAXHOURS.

0 Good
2 Bad: - No Signal
8 Bad: - Signal Too Old
12 Bad: - Initializing
13 Good: - Freezing
14 Good: - Increasing Spike
15 Good: - Decreasing Spike

AZUVLDNUM is included for systems where the full functionality of ASCII character string messages cannot be supported.

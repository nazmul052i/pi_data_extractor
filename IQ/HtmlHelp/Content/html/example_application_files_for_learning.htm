Directories Reference

Purpose

Example application builder files—such as preconfigured models, completed example applications, collections of measurement or vector data, files for tutorials, and so forth—are installed automatically along with the current program.

Important:  The directories listed below contain example files for MULTIPLE Aspen Manufacturing Suite Advanced Process Control products.

To find the location of example application builder files installed on your workstation:

  1. Look in the following directory:

    PUBLICDOCUMENTS_PROGRAMDATA\Examples

To determine which of the example files are applicable to the Builder program you are using:

  1. Use the following guide, which lists application file name extensions, to determine the example files' applicability to specific Builder programs.

Aspen DMCplus

  • .mdl and .ccf - Model and controller configuration files produced by Aspen DMCplus Model and DMCplus Builder.

  • .clc - Vector data for importing.

  • .ind and .dep - Vector data for importing.

  • .xls - Measurement or vector data in Microsoft® Excel™ format for importing.

  • .gif - Graphic illustration files.

Aspen Inferential Qualities (Aspen IQ)

  • .iqf - Inferential sensor files built with IQconfig.

  • .iqm - Model files built with IQmodel.

  • .iqr - Model files exported, or deployed, with IQmodel.

  • .svr - Sensor validation model files.

  • .txt - Measurement or vector data in text file format for importing.

  • .mdl - Aspen DMCplus model file for importing.

Aspen Process Controller

  • .apcapplication - A complete application file produced by Aspen Process Controller (APC) Builder.

  • .apcmodel - A model file produced by APC Builder.

  • .mdl and .ccf - Aspen DMCplus model and controller configuration files for importing.

  • .txt - Measurement or vector data in text file format for importing.

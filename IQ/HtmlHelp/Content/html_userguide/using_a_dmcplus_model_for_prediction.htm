Using a DMCplus Model for Prediction

When using an DMCplus Model in your prediction module, you must complete the following steps.

To configure a prediction module to use a DMCplus model:

  1. After selecting to use a DMCplus model and clicking OK to close the Model Type dialog, the DMCplus Model dialog appears, with three tabs on it: Model, Inputs, and Outputs.

  2. On the Model tab, locate the model file you want to use. Click the Browse button to display an Open dialog. Navigate to the desired path (drive and directory) where the file is stored. Files with an extension of .mdl are shown. Double-click the desired file name. The name appears in the Model File field and a list of all Responses appears.

    Enter values for the Tau 1, Tau 2, Deadtime, and Gain.

  3. Click the Inputs tab. For each input, you can dynamically shift the inputs with a dead-time and two lags (Tau 1 and Tau 2).

  4. An input can also be the result of another calculation, which you specify using the Input Calculation field. You can call up the Function Wizard to help you create a calculation; click the F(x) button.

    As new inputs are entered into the calculation specification, they appear in the Calculation Variables: Values and DCS connections list.

  5. Assign the appropriate source and tagname to the individual inputs in the Calculation Variables: Values and DCS connections list. The CIM-IO Device and Unit # are not required if they are the same as the default device specified in the Config section.

  6. You can create both dynamic and steady-state model outputs. The unbiased steady-state prediction, UPR, is a result of a linear combination of the model input gains. The unbiased dynamic prediction, UAZ, is a result of the dynamic models in the DMCplus model. These values can be written to the database by assigning each a tag name. Click the Outputs tab and set the Cim-IO Device, Unit #, Source, and Tagname, as appropriate, for the Outputs and Results.

  7. Click OK when done. The IQ is created.

    Clicking the Change Type button can change the model type. The UAZ and UPR parameters for the inferred quality are automatically added to the data collection lists.

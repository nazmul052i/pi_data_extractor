Customizing CIM-IO for DMCplus and Aspen IQ

You can customize the way Cim-IO handles time-outs, GET frequencies, and timestamps by using the environment variables and symbols described below.

Time-outs

The Cim-IO time-out can be adjusted by defining an environment variable named CONNECT_CIO_TIMEOUT. The default time-out is 30 seconds. To change it, set the variable to the desired time-out (in seconds) before running Aspen APC Manage or starting a controller.

See the end of this topic for instructions on setting environment variables.

GET Requests

The Cim-IO frequency for GET requests can be specified by defining an environment variable named CIMIO_CIO_xxxxx_FREQ, where xxxxx is the Cim-IO logical device name. The frequency value is specified in seconds. The default frequency value of zero is used when the environment variable is not defined.

Cim-IO for OPC passes this frequency with each GET request when it communicates with OPC Servers. If the frequency is 0.0, the GET request goes all the way to the device to be satisfied. If the frequency is non-zero, the OPC server returns the data to Cim-IO from its cache and sets the list refresh rate to the frequency specified. Specifying zero for the frequency can dramatically reduce the read throughput for an interface.

See the end of this topic for instructions on setting environment variables.

Timestamp Values

The timestamp values returned by Cim-IO can be adjusted by defining an environment variable named CIMIO_CIO_xxxxx_ADJTIME, where xxxxx is the Cim-IO logical device name. The time adjustment is specified as a positive or negative number of minutes and only applies where you have a Cim-IO data type of TIME. The default when the environment variable is not defined is zero minutes (no adjustment).

To set environment variables on Windows systems:

  1. Right-click the My Computer icon on the desktop.

  2. Select Properties and then select the Advanced tab.

  3. Click Environment Variables…. The Environment Variables dialog is displayed.

  4. Under System Variables, click New. The New System Variable dialog is displayed.

  5. Enter the Variable Name and then enter the format string in Variable Value.

  6. Click OK and then verify that the new environment variable has been added in the System Variables window.

  7. Click OK to close the Environment Variables… dialog.

  8. Click OK to close System Properties window.

  9. Reboot the computer to use the new settings.

    Note:  Modifying the value for an existing system environment variable only requires a restart of ACO Base.

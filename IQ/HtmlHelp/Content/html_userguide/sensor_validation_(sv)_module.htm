Sensor Validation (SV) module

Sensor validation in Aspen IQ refers to multivariate model based sensor validation. This is an approach that makes use of redundancy in the plant measurements. Typically, sensor measurements exhibit a correlation structure that can be established by a training procedure using collected or historian data. This correlation structure can be monitored online when the correlation structure is broken a possible sensor fault has occurred. However, the breaking of this correlation structure could also be due to process upset, process transition, or some other reason unrelated to sensor fault. The main objective is to determine if this really is a sensor fault and, if so, to identify the offending sensor. The various phases of sensor validation can be summarized as:

  • Data Conditioning
    This checks the incoming sensor values for bad external status, upper and lower bound violation, frozen signal and spike detection.

  • Detection
    This detects a change in the correlation structure; it may or may not be a sensor fault.

  • Identification
    This determines if this is a sensor fault and identifies the particular sensor.

  • Classification
    This classifies the type of sensor fault - bias, drift, precision loss, or complete failure

  • Estimation
    This estimates the size of the fault that allows reconstruction of the true value and replacement of the faulty value

The following entries can be used to control the execution of some these phases:

SVDETECTONOF

This switch can be used to turn off the detection of change in correlation structure. When this switch is off, sensor faults such as bias, drift, precision loss, or complete failure will not be identified. This switch does not affect the checks performed during data conditioning phase.

SVCLASSIFYONOF

This switch can be used to turn off the classification of sensor faults. This switch purely controls the diagnostic information (for example, SVFAULTTYP) and does not affect the operation of the SV module.

SVREPLACEONOF

This switch can be used to turn off the estimation phase of sensor validation. When this switch is off, sensor faults detected and classified during the earlier phases will show a bad quality for the corresponding output value instead of being replaced by an estimated value using other redundant measurements.

The tuning parameters used in the detection and identification of the above faults are SVSMTHFCTR, SVDTCTCNFDPCT and SVIDNTCNFDPCT. Changes to the tuning parameters will result in new threshold values calculated for the entries: SVFFDITHR, SVNSFSRTHR, SVNQMEANTHR, and SVNVMEANTHR.

SVSMTHFCTR is the coefficient of the smoothing filter used for SVFFDI and for SVNSFSR. This filter is a first order filter. SVSMTHFCTR must greater than or equal to 0 and strictly less than 1. A value close to 1 means heavy smoothing. A value of 0 means no smoothing.

SVDTCTCNFDPCT is the confidence percentage used for calculating the detection index threshold. This calculation is based on confidence limits for the known statistical distribution of the detection index under the hypothesis that no fault is present. SVDTCTCNFDPCT is typically 95% or 99%. Increasing SVDTCTCNFDPCT decreases the probability of false detection, but it also decreases the number of detected faults.

SVIDNTCNFDPCT is the confidence percentage used for calculating the thresholds of the various identification indices, particularly the NSFSR, GLR, NQMEAN, and NVMEAN thresholds. These calculations are based on confidence limits for the known statistical distribution of the indices under the hypothesis that no fault is present. SVIDNTCNFDPCT is typically 99%. Increasing SVIDNTCNFDPCT decreases the probability of false identification, but it may also decrease the number of identified faults and adversely affect the correct classification of the fault type.

Status information for sensor validation model calculations

The following indices provide status information for the sensor validation model calculations:

SVFFDI

Filtered Fault Detection Index. This detection index is monitored and compared with threshold SVFFDITHR. If SVFFDI exceeds the threshold, it means that the expected correlation structure of the data has been broken and that a possible sensor fault has occurred. However the breaking of this correlation structure could also be due to process upset, process transition, or some other reason unrelated to sensor fault.

In order to determine if the detected fault is really a sensor fault a set of "structured residuals" are formed by transforming the residual error in the correlation structure in various ways. Each possible sensor fault has an associated structured residual, which is insensitive to that fault but sensitive to all other faults. Non-sensor faults will affect all structured residuals. Identification of sensor faults is achieved by looking at various indices constructed from the structured residuals and comparing them with confidence limits. If a fault is detected, each index is examined in turn to see if the fault can be identified.

SVNSFSR

Normalized Squared Filtered Structured Residual. In this index, a filter smoothes each structured residual, then squared, then normalized so that its threshold SVNSFSRTHR is common across all structured residuals. This index is calculated, and can be monitored, at every cycle.

SVGLR

Uses a Generalized Likelihood Ratio Test to detect significant changes in the mean of each structured residual. This index is only calculated during fault detection phases. One GLR index is calculated for each structured residual. Statistics are gathered from the point at which fault detection occurred. A fault is identified when the smallest GLR index shows no significant change from zero mean as indicated by the threshold SVGLRTHR.

SVNQMEAN

Attempts to detect changes in the mean of each structured residual, and is calculated only during fault detection phases. SVNQMEAN simply accumulates the mean of each structured residual since the fault occurred and compares it with the threshold SVNQMEANTHR which is calculated on the assumption that the residual is zero mean. A fault is identified when SVNQMEAN is below SVNQMEANTHR for one structured residual, but not for any other.

SVNVMEAN

Attempts to detect changes in the variance of each structured residual, and is calculated only during fault detection phases. SVNVMEAN simply accumulates the variance of each structured residual since the fault occurred and compares it with the threshold SVNVMEANTHR. A fault is identified when SVNVMEAN is below SVNVMEANTHR for one structured residual, but not for any other. This index is good for determining precision loss.

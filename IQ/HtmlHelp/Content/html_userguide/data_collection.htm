Data Collection Module

Note:   The data collection (DC) module is required if you include the prediction, analyzer update, and/or lab update modules in an IQ. For more information, see Creating a Data Collection Module.

Aspen IQ includes a data collection utility, which allows you to create a snapshot of your process data at a specified frequency. This data is then stored in a data collection history file. This file is a circular file, defined to be long enough for a lab sample to be entered into the Aspen IQ system, be returned from the lab, then entered into the Aspen IQ system.

The DC history file stores binary data for a list of points specified in the data collection module. A new value, DCPNTVAL, is written for these specified points during each cycle, using an update frequency specified in the Config module DCUPDFREQ parameter.

The data collection history file includes values for tags configured in the data collection DC module, and for tags specified in the steady-state detection SSD module. In addition to the DCPNTVAL entries, the following values for the inferred quality are also added to the data collection list when you include a prediction and steady-state module in an IQ.

  • UPR  – Unbiased Steady-State Prediction

  • BPR  – Biased Steady-State Prediction

  • UAZ  – Unbiased Dynamic Prediction

  • BAZ  – Biased Dynamic Prediction

  • SS  – Total Percentage Steady State

  • ISS  – Individual Percentage Steady State

The maximum number of historical values stored for each data point is specified in the Config module using the maximum number of records, DCMAXREC, and the data collection frequency in seconds, DCUPDFREQ. For example, if DCMAXREC is set to 1440 and DCUPDFREQ is set to 60, then the file will over-write itself in 24 hours (1440/60). AspenTech recommends that DCMAXREC and DCUPDFREQ be set so that the resulting ratio provides ample time for a lab sample to return from the laboratory.

The data collection update tolerance, DCPCTUPDTOL, is used to control missed update cycle messages. When the time difference between the last update and the current update exceeds the update period, DCUPDFREQ, by more than the tolerance interval, then the message is logged. The tolerance interval is calculated as the data collection update tolerance multiplied by the data collection frequency, divided by 100.

The data collection module calculates a running average of all data collection points, which are stored at a fixed update frequency. The average value is computed using all the data points stored in the history file for a given record. The status of the average value is determined by comparing the percentage of points with good status in the history file with the DCPCTGOOD parameter. The average value is available for output by entering a DCS tag name in the DCAVGOUT entry. Inferential calculations frequently use averaged process values as model inputs; the DCAVGOUT parameter provides a quick way of configuring an averaging calculation.

The DC history file backup option, DCBCKOPT, can be configured to automatically generate backup copies of historical data. The current file information can be copied to the backup file several times each day as specified by the backup frequency in hours parameter, DCBCKHRS. The DCBCKVERS parameter specifies the number of daily backup versions to keep.

Note:  The parameters for defining the data collection frequency, record size, update tolerance, percentage good, and backup generation are specified in the IQconfig section, and apply to all inferential qualities within the IQF application.

Configuration Entries

The following entries are used to configure data collection modules:

DCBCKHRS

Data Collection Backup Frequency

DCBCKOPT

Data Collection History File Backup Option

DCBCKVERS

Data Collection Number of Daily Backup Versions

DCMAXREC

Data Collection Maximum Number of Records

DCPCTGOOD

Data Collection Percent Good Values Required for Average

DCPCTUPDTOL

Data Collection Update Tolerance as Percentage of Update Frequency

DCUPDFREQ

Data Collection Update Frequency

Input Entries

The following entries provide input values to the data collection module:

DCDESC

Data Collection Point Description

DCEU

Data Collection Point Engineering Units

DCLABEL

Data Collection Point History Label

DCPNTSTS

Data Collection Point External Status

DCPNTVAL

Data Collection Point Value

Output Entries

The following entry holds the output values from the data collection module:

DCAVGOUT

Data Collection Point Running Average

What is Aspen Inferential Qualities (Aspen IQ)?

In a process, an inferred quality is a calculation that uses a combination of direct, continuous process measurements to estimate a property that cannot be measured directly and / or continuously.

This property is typically measured using an online analyzer or by taking a sample for laboratory analysis. Online analyzers typically have a cycle time of 5 to 45 minutes, and a laboratory can take anywhere from 30 minutes to 10 hours to provide sample analysis results.

The inferred quality calculation needs to be updated periodically, so that it matches the results from the analyzer and / or laboratory.

The Aspen IQ program, from Aspen Technology, allows you to implement inferred quality calculations without custom programming, in either an open-loop or closed loop system. It includes a common set of options which will satisfy a wide range of applications.

Aspen IQ consists of:

  • IQmodel
    For developing linear PLS, non-linear PLS, Hybrid Neural Net, or Monotonic Neural Net inferential sensor models.

  • IQconfig
    For configuring the input and output connections and setting the initial update and tuning parameters, in a file called the IQF (this file format is consistent with the DMCplus CCF format).

  • Aspen APC Manage
    For loading and managing Aspen IQ applications on an online applications server. See Online Applications Management Overview.

  • IQextract
    For gathering data for use in developing models or general reporting.

Aspen IQ uses Connect from AspenTech to connect with various process databases.

Along with IQmodel files, Aspen IQ also supports Analyzer Predictor models (i.e. Aspen DMCplus model files) and Algebraic models (Multiple Regression Models, Linear PLS Models (non-IQmodel files), Linearized Rigorous Models (DMO inferentials), Custom Models, etc.

The approach of Aspen IQ is that you can create an IQ, or Inferred Quality, composed of various modules, each designed to handle a specific aspect of building and maintaining an inferred quality calculation. An Aspen IQ application consists of one or more IQ calculations.

Monitoring and tuning online applications

The Operations and Engineering views (in the Online tab) of Aspen Production Control Web Server enable you to to monitor and tune online Aspen IQ applications. See Online Applications Management Overview and Web-Based Interface to Online Aspen IQ Applications.

In the online environment, each IQ application runs as two processes:

  • The prediction process (IQpredict)

  • The bias update process (IQupdate)

IQpredict modules

  • Sensor Validation (SV)
    This module executes the sensor validation model generated by IQmodel to identify sensor faults, classify the type of fault, and reconstruct the faulty sensor measurement using other redundant, correlated measurements.

  • Input Calculation (IC)
    This module allows you to implement Aspen Calc based calculations that run before the Prediction (PR) module.

  • Prediction (PR)
    This module is used to calculate a predicted quality based on an algebraic equation, IQmodel, or analyzer predictor model.

  • Output Calculation (OC)
    This module allows you to implement Aspen Calc based calculations that run after the PR module.

  • Steady State Detection (SSD)
    This module monitors variation in process measurements; it ranks values and calculates a weighted average to determine how close the process is to steady state.

  • Process Data Collection (DC)
    This module collects process data values from a DCS and places them in a history file.

IQupdate modules

  • Analyzer Validation and Update (AZU)
    This module adjusts the prediction bias using an online analyzer.

  • Laboratory Data Collection (LDC)
    This module collects lab samples (timestamp and results) in a history file (this file also has information from the DC history file, to show process events, and the SSD, to show if the process is close to steady state).

  • Laboratory Validation and Update (LBU)
    This module adjusts the prediction bias from lab data (using results from the DC, LDC, and SSD modules)

Aspen IQ can also interact with a DMCplus controller, triggering the controller to run after a prediction calculation. Typically, you would use Aspen IQ with a DMCplus controller that includes intermittent CVs (to make them continuous).

For Cim-IO, each source type matches a combination of Cim-IO data type (DT) and Cim-IO device data type (DDT).

The DT determines how the value is actually passed back to Connect. Because Connect treats all non-ASCII values as real, they end up being converted to real, regardless of their DT.

The important data type is the DDT, which is the data type actually requested from the device. Some devices have no requirements about device data type; you can obtain any tag as DBVL (DDT is real).

Other devices (notably Honeywell CM50S and OpenDDA200) require that a given tag is requested in its native format. For these devices, you must specify the correct source to obtain the appropriate DDT. The Cim-IO device manual for each device lists the valid DDTs for that device and how to use them.

In particular, DBST, CBST, and CBAW (the "smart" data types) may have special requirements. Read the specific device manual for more information.

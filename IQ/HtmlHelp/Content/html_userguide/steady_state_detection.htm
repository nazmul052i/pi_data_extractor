Creating a Steady State Detection collection

Validation Overview

The Steady State Detection algorithm allows you to specify a number of key process variables that characterize the state of the process. When all of these variables are steady, then the unit as a whole is considered steady. These values are represented as percentages, with 100% meaning top operation and 0% meaning unsteady, chaotic behavior.

Primary uses of the steady state detector

The steady state detector has four primary uses:

  • Provides additional information in model development for outlier rejection.

  • Provides another optional criteria for lab updating.

  • Can be used to tell operators when to take a lab sample.

The steady-state determination for each variable is calculated using a Heuristic approach, based on the difference between a light filter and heavy filter.

Increasing the heavy filter, SSHVYFILT, for a variable causes its calculated steady-state percentage to return to steady state (100%) more slowly after a disturbance. Decreasing this filter causes the calculated steady-state percentage to return more quickly to steady state.

HVYFILTPV = SSPNTVAL * SSSHVYFILT + (1 – SSHVYFILT) * HVYFILTPV_OLD

The filter factor you set should be as small possible (maximum filtering) while still providing the desired response in the steady-state percentage after a disturbance. The heavy filter factor is clamped between 1.0 and 0.0001.

The light filter factor is calculated from the heavy filter factor and a desired filter factor ratio, SSFFRATIO, between the two filters. By using this ratio, the steady-state detector percentage is more stable than just using a light filter value.

LITFILT = SSHVYFILT * SSFFRATIO

LITFILTPV = SSPNTVAL * LITFILT + (1 – LITFILT) * LITFILTPV_OLD

Increasing SSFFRATIO causes the resulting spike in the steady-state percentage during a disturbance to drop more dramatically. The higher the ratio, the larger the magnitude of the drop in the steady-state percentage when a disturbance is encountered.

The ratio should be set as small possible to provide resistance to routine noise, yet still providing the desired movement in the SS % during an upset. The ratio is clamped between 1.0 and 10.0.

The steady-state determination is based primarily on the ratio of the difference between filtered values and a three sigma-operating limit. There is no statistical significance to this three sigma limit, other than it is a common term for evaluating the quality of a variable, The value for the standard deviation, SSSTDDEV, is a user-specified constant value; the default is 1.0.

After the steady-state determination for each variable has been calculated, the combined steady-state quality is calculated. You can rank the variables, according to their influence on the process's stability. Set the steady-state detection rank, SSRANK, between 0 and 10 (with 10 the most important).

These ranks are used to adjust the steady state percentage for each process variable. A rank of 0 will cause the Aspen IQ algorithm to ignore the effects of the variable. You can also give all variables the same rank. In this case, the steady-state percentage is an average of all the variables.

The steady state detection point value, SSPNTVAL, is available for input validation via the validation section.

SSD Input Entries

The following entries provide input values for each process variable defined in the steady state detection module:

SSDESC

Steady State Detection Point Description

SSEU

Steady State Detection Point Engineering Units

SSFFRATIO

Filter Factor Ration

SSHVYFILT

Heavy Filter Factor

SSLABEL

Steady State Detection Point Label

SSPNTVAL

Steady State Detection Point Value

SSRANK

Steady State Detection Rank

SSSTDDEV

Standard Deviation Value

SSPNTSTS

Steady State Detection Point External Status

SSD Output Entries

The following entry holds the individual percentage steady state value for each process variable:

SSIND

Individual Point Steady State Percentage

The combined steady state percentage entry is found under the IQ:IQNAME folder as:

SSTOTAL

Combined Percents Steady State

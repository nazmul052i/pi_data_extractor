Using Aspen IQ

General steps for using Aspen IQ:

  1. You can set up a data collection, using IQconfig, to collect process, laboratory, and analyzer data for use in developing a model (if you plan to include a prediction or sensor validation module in the IQ).

  2. If you are going to include sensor validation in your application, create a SVR model file using IQmodel.

  3. If you are going to include a prediction in your application, create a model file from any of the following sources:

    • Create an MDL model file by using DMCplus Model (for Analyzer Predictions).
    • Create an IQR model file by using IQmodel.
    • Create an AIQ model file by using Aspen DMC3 Builder, through the Datasets, Inferential Vector, Export feature.
    • Create an .RNN and .PB model files set by using Aspen DMC3 Builder, Nonlinear CVs, Deep Learning Inferential Model, Export feature.
    • Configure Algebraic prediction models directly in IQconfig.

    For more information, see Creating a Prediction Module.

  4. Use IQconfig to create an IQF.

  5. Transfer the IQF and model files to the online system.

  6. Use Aspen APC Manage to load and start the IQF.

  7. Use Aspen Production Control Web Server to view data and make changes to various parameters in the IQF.

    • For more information, see Web-Based Interface to Online Aspen IQ Applications.

  8. Use Aspen APC Manage to save any tuning parameter changes to the IQF so that the changes are not lost during a subsequent reload of IQF.

  9. Use Aspen APC Manage to set the autostart flags for the applications that need to restart automatically following a system reboot.

  10. Extract data using IQextract, for use with other programs. You can create either a CLC format file or CSV (comma separated variable) file.

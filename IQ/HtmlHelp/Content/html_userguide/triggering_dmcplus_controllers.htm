Triggering DMCplus Controllers

You can set an Aspen DMCplus controller to be triggered from an Aspen IQ application.

Expected results

After the IQpredict process has finished its execution, it will trigger the DMCplus controller. The controller will run at the same frequency as the IQpredict process that is specified by the P1PERIOD entry. The DMCPTIMEOUT entry is used to specify the number of seconds the IQpredict will wait for the DMCplus controller to finish its execution cycle and return the results to the IQpredict process.

Important notes and guidelines

  • Generally, you should include all the inferential sensors you want to associate with an Aspen DMCplus controller in one Aspen IQ application.

  • The Aspen DMCplus controller must be running on the same node as the Aspen IQ application.

  • Make modifications in the Aspen IQ application AND in the Aspen DMCplus controller configuration file.

To configure triggering of Aspen DMCplus controllers from an Aspen IQ application:

  1. Select the Config folder, then click the Value field for the entry DMCPNAME.

  2. Type the name of the desired Aspen DMCplus controller in the Value field.

To modify the Aspen DMCplus controller configuration file:

  1. Using Aspen DMCplus Build, open the desired CCF.

  2. Open the Configure section, and modify the entry WTMODE.

  3. Set the value to 2 to enable external scheduling. The controller process waits until an external source signals it to run.

    Note:  Refer to the Aspen DMCplus Entry Dictionary help file for details.

Using IQconfig

You will use IQconfig to configure inferential sensors. With IQconfig, you can develop and deploy inferential sensors on a wide variety of computing platforms.

Most of the tasks you will perform in IQconfig involve managing and setting entry information.

IQconfig creates a file called an IQF, which contains one Aspen IQ application. This Aspen IQ application can contain one or more inferential sensors, also called inferred qualities (IQ).

Modules

Each inferred quality (IQ) consists of up to nine (9) modules:

  • Sensor Validation (SV)

  • Input Calculation (IC)

  • Prediction (PR)

  • Output Calculation (OC)

  • Steady-state Detector (SSD)

  • Data Collection (DC)

  • Analyzer Update (AZU)

  • Lab Data Collection (LDC)

  • Lab Update (LBU)

You can select whether the IQ will contain a prediction, an analyzer update, and/or a lab update.

Selecting a model type

If you choose to include a prediction, you must specify a model type and model source. IQconfig supports the following inferential sensor model types:

  • Analyzer Predictors

    • DMCplus Model files

  • IQmodel Model files

    • Linear PLS Model

    • Non-linear PLS Model

    • Hybrid Neural Net Model

    • Monotonic Neural Net Model

  • Algebraic Models

    • Multiple Regression Models

    • Linear PLS Models (Non-IQmodels)

    • Linearized Rigorous Models (DMO inferentials)

    • Custom Models

If you choose to include sensor validation, you must specify a model file created using IQmodel.

Calculation variables can be mapped to a number of different variables from within the IQ application. This is useful if the inputs to a calculation are calculated by other modules within the IQ. For example, the second IQ in the application may make use of the biased prediction (BPR) value from the first IQ in the application. Similarly, the output from a calculation may be mapped to other variables in the application. For example, if a detailed calculation is used to determine an input value that is used by more than one IQ, then it can be calculated once and then mapped to downstream Prediction Module calculation variables (CVARs). The mapping capability effectively allows the chaining of inferentials.

Note:  The Prediction Modules of the IQs in the application are run in the order that they appear in the tree view in IQconfig and in online views of the IQ application (in APC Web Server). For mapping to be effective, ensure that you create the IQs in IQconfig in the order you want them to run.

To configure each of the inputs and outputs to map to or map from another IQ variable:

  1. In the Calculation Properties dialog box, select the input or output.

  2. Set the Keyword to LOCAL.

  3. Click the â€¦ button in the Map From column. The Input Parameter XX mapping dialog box is displayed, where XX is the name of the variable.

  4. Select an entry using the IQ Name, Module, and Entry list boxes:

    1. Click for IQ Name to display the available IQs. Then click an IQ to select it.

    2. Click for Module to display the modules available for the selected IQ. Then click a module to select it.

    3. Click for Entry to display the variables available for the selected module. Then click a variable to select it.

  5. Click OK. The information is displayed in the Calculation Properties dialog box.

  6. Click OK to close the dialog box. From this point on, the selected variable will be mapped from the specified map variable.

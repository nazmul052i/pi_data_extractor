Configuring Validation for Variables

To configure validation for variables:

  1. Open Aspen IQ.

  2. Locate the Validation folder.

  3. Right-click the Validation folder to display the context menu.

  4. Select the Properties option. The Validation Properties dialog box is displayed.

  5. Select an entry using the IQ, Module, and Variable list boxes:

    1. Click for IQ to display the available IQs. Then click an IQ to select it.

    2. Click for Module to display the modules available for the selected IQ. Then click a module to select it.

    3. Click for Variable to display the variables available for the selected module. Then click a variable to select it.

  6. Click Add. The selected variable is added to the list of variables in the middle panel.

    Note:  The Remove button allows you to remove a variable from the list.

    Variables that have already been configured for validation will display Yes in the Enabled column. When you select a variable, the validation information for that variable is displayed in the DataValidation panel. You can edit this information.

  7. To set/edit the lower limit:

    1. Select the Lower limit check box.

    2. Enter the limit in the Low limit value: box.

    3. Select the Mark bad or Clamp radio button. This will set the default behavior for the value when it is below the low limit. (That is, it will either clamp out of range values to the low limit or mark the value as Bad.)

  8. To set/edit the upper limit:

    1. Select the Upper limit check box.

    2. Enter the limit in the Upper limit value: box.

    3. Select the Mark bad or Clamp radio button. This will set the default behavior for the value when it is above the upper limit.(That is, it will either clamp out of range values to the upper limit or mark the value as Bad.)

  9. To set/edit the flatline detection:

    1. Select the Flatline detection check box.

    2. Enter the freeze deadband value in the Flatline threshold: box. If the absolute value of the current value minus the previous value does not exceed the freeze deadband, then the signal is considered frozen.

    3. Enter the maximum freeze time (in minutes) in the Flatline period box. When the signal has been frozen for longer than the flatline period, then the data will be considered bad.

  10. To set/edit the spike detection:

    1. Select the Spike detection check box.

    2. Enter the rate of change threshold in the Spike threshold: box. If the absolute value of the current value minus the previous value exceeds the spike threshold, then the signal is considered bad.

    3. Enter the minimum out of range value (in minutes) permitted for spiked data in the Spike period box. If a spike has been detected and the signal remains out of range for longer than the spike period, then the signal is considered at a new operating region and will be marked good.

  11. To use a default value if the raw value fails any of the validation tests, then select the If bad, use default check box. This option will override the high and low limit 'if bad' behavior.

    Note:  By using the interface provided by Aspen Production Control Web Server, you can change the value of any of the various validation entries after the IQ application is placed online. (See details in the last step, below.) However, only the types of validation that have been enabled in IQconfig will be executed online. For example, if low limit checking is enabled via IQconfig, then the value for VVARMINLIM can be changed online. Conversely, if high limit checking is not enabled, then values entered online, in Aspen Production Control Web Server, will be ignored.

  12. Click OK to save the input and close the dialog box. For each variable that was added in the Validation Properties dialog box, there will be a corresponding folder listed under the VALIDATION folder.

  13. Save the iqf file.

  14. Use Aspen APC Manage to load the new .iqf into the online system.

  15. To confirm validation entries and results online, use the online interface provided by Aspen Production Control Web Server. (Also see Web-Based Interface to Online Aspen IQ Applications.)

    1. In Aspen Production Control Web Server (PCWS), Online tab, navigate to the Overview page.

    2. In the Overview table that lists Aspen IQ applications (and their IQs), click the name of the application, as illustrated below.

      As a result, an IQ Application Detail dialog box is displayed.

    3. In the IQ Application Detail dialog box, scroll to the Validations section, which is at the bottom of the dialog.

      Note that all variables for which validation is enabled are listed in the Validations section.

    4. In the Validations section, click any of the links under the Status column to display a Validation Detail dialog box.

      Note:  You will be able to view the raw value and validated status in the Validation Detail dialog box. VVARVLDOPT is set by IQconfig and is based on how you configured the validation for a given input or output. The IQ online engine uses this value to enable or disable the various validation checks.

    5. Click any item in blue, underlined text to make edits if you need to revise validation criteria. A dialog box is displayed for editing.

    6. In some instances, the IQ application panels displayed in the Engineering view of Aspen Production Control Web Server also provide links that enable you to access and revise validation entries:

      • In the upper portion of the panels, look for validation status messages that display "Good," "Good: Clamped Low," "Good: Clamped High," "Below Min Limit," and so forth. (See illustration below.)

      • Click any status message displayed in blue, underlined text to review the validation criteria in a Validation Detail dialog box and, if necessary, make edits.

Using an Algebraic Model for Prediction

When using an Algebraic Model in your prediction module, you must complete the following steps.

To configure a prediction module to use an algebraic model:

  1. After selecting to use an Algebraic Calculation Model and clicking OK to close the Model Type dialog, the Algebraic Calculation Model dialog appears, with three tabs on it: Model, Inputs, and Outputs.

  2. On the Model tab, enter a Calculation in the field. Or, you can click the F(x) button to display the Function Wizard, which you can use to build a calculation.

    Once you enter the calculation, the input variables are parsed into the List of Inputs box below and will appear on the Inputs tab (MVAR) and the Outputs tab (CVAR).

  3. Enter values for the Output Filters, Tau 1, Tau 2, and Deadtime.

  4. On the Inputs tab, each input can be delayed and filtered with up to two lags and a dead-time, Tau 1 (INFILT1), Tau 2 (INFILT2), and Deadtime (INFILTDT), respectively. These parameters exist for each MVAR and can be later modified in IQconfig or in an online view of the application (in APC Web Server).

  5. Along with the filtering and delay, an additional calculation for this input can be specified in the Input Calculation field. You can call up the Function Wizard to help you create a calculation; click the F(x) button.

    As new inputs are entered into the calculation specification, they appear in the Calculation Variables: Values and DCS connections list.

  6. Assign the appropriate source and tagname to the individual inputs in the CVAR Entries list. The CIM-IO Device and Unit # are not required if they are the same as the default device specified in the Config section.

  7. As a new input variable is created, it is added to the Calculation Variables: Values and DCS connections list. The Calculation Variables: Values and DCS connections list is where you can enter the source and tag name (as well as the Cim-IO Device and Unit #, if desired) for each input variable entry.

  8. The Outputs tab is where you specify the output tag name for the unbiased steady-state prediction, UPR, and the unbiased dynamic prediction, UAZ. If no dynamic prediction is desired, then this output can be ignored. A dynamic prediction can be produced via a filtering and delay by entering up to two lags and a dead-time in the Output Filters box on the Model tab. The unbiased steady-state prediction is passed through these dynamics to create the unbiased dynamic prediction. Finally, any model variable calculation results (after applying the optional input filters) can be written to external tags, by specifying the desired source and tagname in the appropriate fields in the Results box on the Output tab.

  9. Click OK when done. The IQ is created.

    Clicking the Change Type button can change the model type. The UAZ and UPR parameters for the inferred quality are automatically added to the data collection lists.

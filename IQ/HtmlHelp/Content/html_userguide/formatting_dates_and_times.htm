Formatting Dates and Times

Aspen APC Manage formats date and time strings for display using the standard ANSI C function strftime. The way these dates and times are formatted (including the language used for month names) can be changed using an environment variable.

Changing the date and time formats

To change the format of the date and time, you must have a copy of the standard ANSI C library reference that has a full list of formatting conversion specifiers for the strftime C function. Use this reference when setting the characteristics of the date and time format variables.

The following variable is available to you for setting the date and time formats:

  • ACOD_DATTIM_FMT
    Determines the format of the date and time string displayed when messages or controller applications are listed in Aspen APC Manage.

    Valid values are:

    • %a – abbreviated weekday name

    • %A – full weekday name

    • %b – abbreviated month name

    • %B – full month name

    • %d – day of the month (01-31)

    • %H – hour (24-hour clock) (01-24)

    • %I – hour (12-hour clock) (01-12)

    • %M – minutes (00-59)

    • %p – local equivalent of AM/PM

    • %S – seconds (00-59)

    • %Y – 4-digit year with century

To set environment variables on Windows systems:

  1. Right-click the My Computer icon on the desktop.

  2. Select Properties and then select the Advanced tab.

  3. Click Environment Variables…. The Environment Variables dialog is displayed.

  4. Under System Variables, click New. The New System Variable dialog is displayed.

  5. Enter the Variable Name and then enter the format string in Variable Value.

  6. Click OK and then verify that the new environment variable has been added in the System Variables window.

  7. Click OK to close the Environment Variables… dialog.

  8. Click OK to close System Properties window.

  9. Reboot the computer to use the new settings.

    Note:  Modifying the value for an existing system environment variable only requires a restart of ACO Base.

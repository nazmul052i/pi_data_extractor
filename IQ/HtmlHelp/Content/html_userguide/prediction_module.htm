Using a DMCplus Model for prediction

When creating a new IQ, you can choose to add a prediction, based on an algebraic equation, an IQmodel file, or an Analyzer Predictor model (DMCplus model).

The following model types are supported:

  • Algebraic Models

  • Multiple Regression Models

  • Linear PLS Models (Non-IQmodel files)

  • Linearized Rigorous Models (DMO inferentials)

  • Custom Models

  • IQmodel Files

  • Linear PLS Model

  • Non-linear PLS Model

  • Hybrid Neural Net Model

  • Monotonic Neural Net Model

  • Analyzer Predictors

  • DMCplus Model Files (FIR Models)

If you include a prediction in your IQ, you must also create data collection and steady-state detection module sections.

IQconfig will automatically detect the independent model variable by reading the model file or parsing the algebraic model equation. By definition, the prediction models will contain only one dependent variable. We will refer to the independent variables as model variables (MVAR) and the dependent variable as the prediction value.

The algebraic expression MVARSRC can be specified for each model variable. The individual variables within the algebraic expression can be connected to DCS tags using the CVARPNTVAL entries. Alternately, the model variable itself can be directly connected to a DCS tag.

A second order plus deadtime filter is applied to the model variable input value before it is used in the model calculation. This input filtering can be disabled by settling the two lag constraints, INFILTT1 and INFILTT2, to zero.

The ISDYNAMIC entry indicates the prediction model type as dynamic or steady state. Dynamic models can generate a steady state prediction (UPR) and a dynamic prediction (UAZ). The steady state models can only generate a UPR output.

For steady state models, a second order plus deadtime filter can be applied to the UPR output to generate the dynamic prediction (UAZ). As before, this output filtering can be disabled by setting the two lag constraints, OUTFILTT1 and OUTFILTT2, to zero.

The prediction bias term PREDBIAS is added to the unbiased prediction outputs UPR and UAZ to obtain the biased steady state prediction (BPR) and the biased dynamic prediction (BAZ) respectively.

The source for the prediction bias can be set to a manual entry, i.e., entered by the user, or set to be calculated based on analyzer feedback from the AZU module, or set to be calculated based on lab feedback from the LBU module. The source for the prediction bias can also be set to use both the analyzer and lab feedback.

The update source for prediction bias is specified using the UPDSRC entry, which accepts the following values:

0

Manual entry

1

Analyzer

2

Lab

3

Both Analyzer and Lab

When the update source is specified as Both Analyzer and Lab, the prediction module will preferentially use the analyzer and switch to lab update only when the analyzer bias status is bad. The analyzer bias status can become bad when the analyzer is offline for maintenance or calibration requirements. Similarly, the lab bias status can become bad if the operators or lab technicians stop entering lab samples into the system for a prolonged period of time. When both analyzer and lab bias statuses become bad, the update source automatically switches to Manual Entry. Also, when the lab or analyzer bias status becomes good again, the update source will automatically switch back to the user-specified preferred update source. The bias update source currently used by the prediction module is indicated by the UPDSRCUSED entry.

The prediction initialization flag PREDINIT allows the user to initialize an inferential prediction without stopping the application.

You can apply a standard transform to the outputs of a PR module. The XFORM parameter holds the transform string. For more information on the transform variables that allow you to set the desired transforms, see Transform Overview.

Input validation for the MVAR and CVAR variables can be configured in the validation section. After the prediction is calculated, the following outputs are available for output validation via the validation section:

  • BPR

  • BAZ

  • UPR

  • UAZ

  • PREDBIAS

Prediction (PR) Module Configuration Entries

The execution of the prediction models is controlled by the P1PERIOD entry. The CNTDWN entry can be used to implement a watchdog in the DCS that will set the quality of the prediction output to Bad in the event of a failure of the IQpredict process or loss of communication between the IQ Online processes and the DCS. The CNTDWN entry writes a value of 2.5 times the P1PERIOD during each execution cycle of the IQpredict process.

P1PERIOD

IQpredict process execution period in seconds

CNTDWN

Watchdog Timer Reset

Connecting Validation Variables to a Database

To connect validation variables to a database:

  1. Complete all the steps in Configuring Validation for Variables.

  2. Browse to the validation variable folder in Aspen IQ tree view.

  3. Select the variable that you want to connect to a database.

  4. Set the Keyword to READ or WRITE.

  5. Enter information in the following columns:

    • Cim-IO Device (A default option is provided.)

    • Unit #

    • Source

    • Tagname

    From this point on, the variable will read from or write to a database.

Validation Input Entries

The following validation variables can be read from a database:

VVARMINLIM

Validation Variable Minimum Limit

VVARMAXLIM

Validation Variable Maximum Limit

VVARROCLIM

Validation Variable Rate of Change Limit

VVARMINTIMOR

Validation Variable Minimum Time Outside Range

VVARFRZDB

Validation Variable Freeze Deadband

VVARMAXTIMFZ

Validation Variable Maximum Freeze Time

VVARDEFVAL

Validation Variable Default Value

Validation Output Entries

The following validation variables can be written to a database:

VVAROUTVAL

Validation Variable Value

VVAROUTSTS

Validation Variable Value Status

VVARVLDSTS

Validation Variable Validation Status

VVARTIMOR

Validation Variable Time Outside Range

VVARTIMFZ

Validation Variable Freeze Time

VVARLASTGOOD

Validation Variable Last Good Value

Validation Configuration Entries

The following variables are used by Aspen IQ to store the validation configuration:

VVARRAWVAL

Validation Variable Raw Value

VVARRAWSTS

Validation Variable External Input Status

VVARVLDOPT

Validation Variable Option

VVARLABEL

Validation Variable Label

Note:  VVARVLDOPT is set by IQconfig and is based on how you configured the validation for a given input or output. The IQ online engine uses this value to enable or disable the various validation checks. The online interface of Aspen Production Control Web Server (PCWS) enables you to change the value of any of the various validation entries. (For details, see the last step in Configuring Validation for Variables.) However, only the types of validation that have been enabled in IQconfig will be executed online. For example, if low limit checking is enabled via IQconfig, then the value for VVARMINLIM can be changed online using PCWS. Conversely, if high limit checking is not enabled, then values entered in PCWS will be ignored.

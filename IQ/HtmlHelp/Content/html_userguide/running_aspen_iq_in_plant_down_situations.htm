Running Aspen IQ in Plant Down Situations

When a plant is down, you may find it helpful to run Aspen IQ in hibernation mode. Setting the hibernation mode to On is similar to issuing an "iq_manage_stop" command. Even though Aspen IQ is still running, much of its functionality is turned off and no data will be collected. Valuable history is not filled up with unnecessary information. When youâ€™re ready to resume normal operation, set the hibernation mode to Off.

Hibernation mode can be set at the application level or the inferential level by entering a value for HIBERNATE or IQHIBERNATE in IQconfig.

You can also set the hibernation mode in the online view of Aspen Production Control Web Server. For details, see Setting Hibernation Mode.

Setting the hibernate flag to On

When you set the hibernation mode to On, at either the application or inferential level, the following occurs:

  • A message will be issued indicating that the application or inferential is hibernating.

  • Debug files generated using P1DBGSW and P2DBGSW will provide information on which inferentials and modules are hibernating.

  • The SV module will not run and the SVOUTSTS will be set to BAD.

  • The PR module will not compute predictions and the bias prediction status (BPRSTS) will be set to zero.

  • The IC and OC modules will run. This is to enable the setting of the hibernation flags based on process conditions. Care should be taken when mapping inputs from other inferentials that are hibernating, as the values may be BAD or stale. One strategy is to include the hibernation switches as inputs to the calculations so that staleness checks can be made.

  • The SSD module will set both the individual steady state percentage (SSIND) and the combined steady state percentage (SSTOTAL) to zero.

  • The DC module will not historize points belonging to hibernating inferentials.

  • The AZU module will ignore new analyzer values and will reset the AZNEWFLAG to zero if necessary.

  • The LDC module will ignore new lab samples and will reset the SAMPFLAG, AVLFLAG, and CNCLFLAG to zero if necessary.

  • The LBU module will ignore new lab samples and will not adjust the bias.

  • The IQ application will still trigger a DMCplus controller if it is configured to do so; however, the BPRSTS will be set to BAD. The controller should be configured to react accordingly.

Setting the hibernate flag to Off

When you set the hibernation mode to Off, at either the application or inferential level, it may be desirable to initialize the predictions (PREDINIT) or the lab bias (LBUINIT), depending on the duration and circumstances of the hibernation.

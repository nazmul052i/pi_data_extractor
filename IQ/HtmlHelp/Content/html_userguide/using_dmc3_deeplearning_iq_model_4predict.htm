Using a DMC3 Deep Learning Inferential Model for Prediction

To configure a prediction module to use a Deep Learning Inferential model that originates from Aspen DMC3 Builder:

  1. After selecting to create a Prediction module (see Creating a Prediction Module), in the Model Type dialog box, select the Deep Learning Model option, and click OK.

    The Deep Learning Model dialog box is displayed. (See illustration below.)

  2. Click the file selection button for Deep Learning Model File. A file Open dialog is displayed.

  3. In the file Open dialog, set the option for files-type to Deep Learning Files (*.rnn). Then locate and select the desired .RNN file, and click Open.

    As a result, parameters for the selected Deep Learning Inferential model files (which consist of a .RNN file and corresponding .PB file) are displayed in the Deep Learning Model dialog box, as illustrated in the example below.

    Note: Some model parameters may require manual entry. See Important note below.

  4. Click the Outputs tab. Enter values for the Output Filters, Tau 1, Tau 2, and Deadtime.

  5. Click the Inputs tab. For each input, you can dynamically shift the inputs with a Deadtime and two lags, Tau 1 and Tau 2. (See Important note below.)

    Important: A Deep Learning Inferential model identified (in Aspen DMC3 Builder) with the MLP option for Algorithm may include Filter and Deadtime settings that were specified during the model identification. When you prepare the IQ configuration (as described in this topic), it is important to replicate (by manually entering) the same values for each of the Inputs in the Tau 1 and Deadtime columns. An example is illustrated above. For other Deep Learning Algorithm options (i.e., GRU or LSTM), you can leave the Tau 1 and Deadtime values set to 0 (zero) to ensure the Deep Learning model operates, in the IQ, in the same manner that was exhibited (in Aspen DMC3 Builder) during model identification validation. For more information, see "Models Identification - Deep Learning IQ" | "Creating a Deep Learning Inferential Model Application" in the Aspen DMC3 Builder Help.

  6. An input can also be the result of another calculation, which you specify using the Input Calculation field. You can call up the Function Wizard to help you create a calculation; click the F(x) button.

    As new inputs are entered into the calculation specification, they appear in the Calculation Variables: Values and DCS connections list.

  7. Assign the appropriate source and tagname to the individual inputs in the Calculation Variables: Values and DCS connections list. The CIM-IO Device and Unit # are not required if they are the same as the default device specified in the Config section.

  8. You can create both dynamic and steady-state model outputs. Any dynamic models that are specified in IQmodel will automatically be specified to the unbiased dynamic prediction, UAZ, parameter and the unbiased steady-state prediction will be sent to the UPR parameter. These values can be written to the database by assigning each a tag name. Click the Outputs tab and set the Cim-IO Device, Unit #, Source, and Tagname, as appropriate, for the Outputs and Results.

  9. Click OK when done. The IQ is created.

    Clicking the Change Type button can change the model type. The UAZ and UPR parameters for the inferred quality are automatically added to the data collection lists.

IQconfig FTP Transfer

IQconfig can transfer your IQF and associated model files directly to an Aspen IQ Online host using FTP. Your Aspen IQ Online host must have the FTP service running and you must have a valid FTP account. You will need the following information in order to connect to your Aspen IQ Online host.

  • Server name
    The TCP/IP host name or the TCP/ IP address of the Aspen IQ Online host.

  • User name
    A valid user name for the FTP login.

  • Password
    A valid password for the FTP login.

  • App directory
    The base Aspen IQ applications directory.

    Please note that this path will be relative to the path you are at when you login to the FTP server. If IQconfig cannot change to this directory an error message will be displayed.

Using IQconfig FTP transfer

The following screen appears after you click Next.

IQconfig will attempt to find the IQF file you are working with and then it will try to find any associated model files. (*.IQR, *.MDL). The status column displays if the file was found or not. For all files that are found you may uncheck the Transfer column entry of that file to stop it from begin transferred to the online host. The location column shows the physical path where the file is located. The application name defaults to the name of the IQF without the .IQF extension attached. You may override this if the application name is different from the IQF filename.

If you wish to override the location path for a file then click the button next to the location column.

If the application directory is not found IQconfig will prompt you to create it.

IQconfig will also prompt you if it is going to overwrite any files within the application directory.

Transform Overview

Transforms are associated with Prediction (PR) outputs. Any PR module output can have a transform associated with it. The parameter XFORM, which is associated with each PR module, will hold the transform string. Aspen IQ will shadow the XFORM variable to preserve the original values.

Note:  When you select a transform type, the lab or analyzer update method must be traditional. Aspen IQconfig will issue a warning on validation, if cusum or scores is chosen for either AZUPDMETH or LBUPDMETH. In addition, the onlines will automatically switch the update method to traditional and log a message if cusum or scores is chosen.

Note:  You can define the transform for a PR module in the Configure Output Transformation dialog box.

How and when transforms are applied

It is important to note that a transform is applied to the Prediction (PR) module output.

The bias is calculated in the transform domain. In addition, the calculated bias is normally filtered into the bias prediction in the transformed domain. However, when the update source is set to manual, the prediction bias is filtered in the engineering domain.

For transformed variables, validation and bias calculation parameters must be considered relative to the current value of the lab or analyzer. A corresponding percent of change is used rather than the raw value in engineering units. This ensures that a single value of the parameter is valid over the entire range of the transformed space. The percent of change is calculated by taking the user entered value of the parameter and dividing by the range specified by the lab or analyzer minimum and maximum limits.

Configuring Validation for Variables

Connecting Validation Variables to a Database

The validation functionality allows you to configure input and output validation on variables. Configuring validation will help prevent the propagation of bad calculations through the program. For example, a measurement that spikes for one cycle may dramatically change the prediction. However if you are allowed to validate the calculation (for example, set limits on it or clamp it), you will be able to prevent abnormal data from influencing the calculation.

Note:  Input validation is triggered before the appropriate module uses the variable for calculations. Output validation is triggered after the appropriate module has been run. For example, the validation in MVAR and CVAR variables is triggered before the prediction is calculated. The validation on BPR, BAZ, UPR, UAZ, and PREDBIAS, however, is triggered after the prediction is calculated.

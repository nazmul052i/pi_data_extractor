Set End Time for History Retrieval

Choose this option to change the end time of the data extracted when the command to generate a CLC or CSV format output file is issued. When the history file is opened, the end time for extraction is set to the time of the latest data stored in the history file. The time of the latest data stored among the list of tags selected for extraction can be viewed using the menu option List start and end times for extract.

The end time must be specified using the fixed format specification string: %m-%d-%Y %H:%M:%S. Please refer to the ANSI C specification of the strftime function for a description of the format specifiers.

The IQextract program is designed to extract one value just outside the specified start or end time range when the start or end time lies between two history record timestamps. This is a design feature provided to ensure that a valid non-zero set of records are returned even when there are no data records between the specified start and end times.

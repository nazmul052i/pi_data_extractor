Analyzer Update (AZU) Module

See Also

Creating an Analyzer Update

Transform Overview

The analyzer update module can be configured for an IQ to perform the following tasks:

  • Analyzer detection
  • Analyzer validation
  • Analyzer bias calculation

Analyzer Detection

Aspen IQ can accept new analyzer values with or without a new signal flag. If the new analyzer value specification option, AZNEWOPT, is set to 1, then the DCS sets the new analyzer flag value, AZNEWFLAG, to1 that indicates that a new analyzer value is available in the NEWANZ entry. NEWANZPNTSTS is the external status of new analyzer value used for analyzer bias updates. Aspen IQ will reset the AZNEWFLAG parameter to zero after reading the analyzer value.

When AZNEWOPT parameter is set to 2, then the analyzer signal is monitored for changes exceeding the freezedeadband, AVFRZDB. An analyzer update is forced if a new analyzer signal is not detected within the forced update interval limit, AZNEWITVLMAX (in minutes).

After an analyzer update is detected, the previous analyzer samples array is shifted down to make room for the new analyzer sample values in sample #1. Online views of the IQ application can display up to fifty previous analyzer updates.

The raw analyzer value is copied from the NEWANZ entry and stored in the PAZURAZ entry for the current sample.

Frequently, the raw analyzer value has a somewhat constant offset from the corresponding lab value. This offset can occur because the analyzer and lab samples are drawn from different locations and/or due to different calibration techniques used for the two measurements. The AZULABOFST, which is added to the raw analyzer value PAZURAZ, can be used to account for this offset. The result of PAZURAZ + AZULABOFST is stored in the AZUVAZ entry and referred to as the validated analyzer value.

A fraction of analyzer drift correction factor, ANZRDRIFT, calculated by the lab update (LBU) module, AZUDRFTFRAC, can be applied to the current validated analyzer signal, PAZUVAZ, using the drift correction factor, AZUDRFTFRAC. This factor can be set to zero to eliminate any drift correction. A factor of 1.0 ensures a smooth transition when switching between lab and analyzer bias update sources. This factor should be used in conjunction with the DRFTNUMVALS to prevent any unwanted perturbations to the analyzer bias.

The current validated analyzer value, AZUVAZ, is calculated as follows:

PAZUVAZ = PAZURAZ + PAZULABOFST

The current drift corrected analyzer value, AZUCAZ, is calculated as:

PAZUCAZ = PAZUVAZ + (ANZRDRIFT * AZUDRFTFRAC)

The analyzer bias calculation requires the unbiased and biased prediction values corresponding to each analyzer update. The user can choose either the dynamic prediction values (UAZ and BAZ) or the steady state prediction values (UPR and BPR) using the AZUDYN entry. The prediction values for the current analyzer signal are stored in PAZUUAZ and PAZUBAZ entries.

The entries AZURAZ, AZUVAZ, AZUCAZ, AZUUAZ, and AZUBAZ represent analyzer sample values corresponding to last good analyzer sample and are updated only when the validation status for the corresponding PAZURAZ, PAZUVAZ, PAZUCAZ, PAZUUAZ, and PAZUBAZ entries for the current sample are good.

The new valid analyzer output flag, AZUFLAG, is only set when the analyzer signal passes all the validity checks. Since Aspen IQ handles all of the bias updating, under normal circumstances AZUFLAG will not be needed. However, there can be a case where Aspen IQ is used only for analyzer signal validation and a DMCplus controller receives the validated analyzer as an intermittent CV. AZUFLAG can then be used to signal the controller of CV updates.

Analyzer Validation

The raw analyzer value is tested against minimum and maximum limits, AVMINLIM and AVMAXLIM, respectively. The analyzer validation compares the change in subsequent analyzer values with the freezedeadband, AVFRZDB, to determine the frozen signal status of the analyzer.

If the analyzer remains frozen beyond the maximum freeze time specified in AVMAXTIMFRZ, then the analyzer value is flagged as frozen and bad. The frozen signal detection is bypassed by the analyzer validation when the freeze dead band is less than or equal to zero.

Aspen IQ will ignore all analyzer updates detected via AZNEWFLAG or AZNEWITVLMAX, while the analyzer is frozen and changes in the analyzer value do not exceed the freezedeadband.

Increasing or decreasing spikes are screened by the increasing and decreasing analyzer rate of change limits, AVROCLIMI and AVROCLIMD, respectively. If a signal decreases lower than AVROCLIMD, then it is flagged as a decreasing spike and given a Bad status (a similar action occurs for increases above AVROCLIMI).

If the signal remains outside the rate of change limit, then the increasing or decreasing time signal is outside of range counters are incremented, AVTIMORD and AVTIMORI, respectively.

The spike detection is bypassed if the analyzer rate of change limit is less than or equal to zero.

If the analyzer signal has changed by a value greater than the increasing or decreasing rate of change limits, but remains at the same level for a period longer than the minimum time limit specified by AVMINTIMOR, (AVTIMORD >= AVMINTIMOR, or AVTIMORI >= AVMINTIMOR), then it is assumed that the process has shifted to a new operating level. In this case, all the previous signals that were flagged as spikes in the same direction are changed to a good status and can now be used for bias update.

When the analyzer signal status remains bad beyond a specified maximum number of hours for analyzer bias update, AVMAXHOURS, the analyzer status is set to bad.

This check is bypassed by setting the AVMAXHOURS value to less than or equal to zero.

The analyzer drift, ANZRDRIFT, calculated by the lab update (LBU) module, is checked against the maximum drift limit, AVMAXDRIFT, to determine the validity status of the analyzer. The maximum drift check is bypassed during analyzer validation if the analyzer drift limit is less than or equal to zero.

The new analyzer bias value, ABIASNEW, is available for validation via the validation section.

Analyzer Bias Calculation

The analyzer bias calculation uses the previous drift corrected analyzer sample values, PAZUCAZ, and their corresponding unbiased and biased predictions, PAZUUAZ and PAZUBAZ, to calculate a raw bias value PABIASRAW. The raw bias value is calculated as the difference between an average of previous drift corrected analyzer values, PAZUAVGCAZ, and an average of corresponding unbiased prediction values, PAZUAVGUAZ.

PABIASRAW = PAZUAVGCAZ - PAZUAVGUAZ

The number of analyzer values averaged to obtain PAZUAVGCAZ depends on the AZUNUMVALS entry and may vary depending on the bias update method specified in the AZUPDMETH entry. The same bias update methods are available for both lab and analyzer bias calculations. These methods are explained in the lab update (LBU) module section later in this chapter.

A fraction of the change in prediction bias recommended by the raw bias value, PABIASRAW, is applied to the existing prediction bias and stored in the new analyzer bias value, PABIASNEW. The fraction is specified using the ABIASFRAC entry:

PABIASNEW = PREDBIAS + (PABIASRAW PREDBIAS) * ABIASFRAC

The analyzer values and the corresponding prediction values used in analyzer bias calculation must have a "Good" status. If there is an insufficient number of previous analyzer samples with "Good" status, then the bias update calculation will fail and the bias update message, PABIASNUM, will be set accordingly.

A successful bias calculation will cause the values of the ABIASNEW, ABIASRAW, AZUAVGCAZ, AZUAVGUAZ, and AZUAVGBAZ entries to be updated from the corresponding PABIASNEW, PABIASRAW, PAZUAVGCAZ, PAZUAVGUAZ, and PAZUAVGBAZ entries for the current sample. The analyzer bias update time entry, ABIASTMSTR, will be updated to the current time.

If there are no successful bias updates for a period exceeding the number of hours specified in AVMAXHOURS, then the analyzer bias status, ABIASPNTSTS, will be set to bad.

AZU history file

The analyzer bias update history file stores the following information for each inferential quality when a new analyzer is available:

RAZ

Raw Analyzer Value

AVS

Analyzer Validation Status (for internal use only)

VAZ

Validated Analyzer Value

CAZ

Drift Corrected Analyzer Value

IUAZ

Unbiased Analyzer Prediction

IBAZ

Biased Analyzer Prediction

ABS

Analyzer Bias Update Status (for internal use only)

ARB

Analyzer Raw Bias

ANB

Analyzer New Bias

CAZA

Drift Corrected Analyzer Value Average

UAZA

Unbiased Analyzer Prediction Average

BAZA

Biased Analyzer Prediction Average

The maximum number of lab records stored in the AZU history file is specified using the AZUMAXREC parameter. The AZU history backup option, AZUBCKOPT, can be configured to automatically generate backup copies of historical data. The current file information can be copied to the backup file several times each day as specified by the backup frequency in hours parameter, AZUBCKHRS. The AZUBCKVERS parameter specifies the number of daily backup versions to keep.

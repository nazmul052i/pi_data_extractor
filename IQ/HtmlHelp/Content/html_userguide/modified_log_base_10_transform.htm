Modified Log Base 10 Transform

See Also

Transform Overview

This is a modified common log transform. Low is the low clamp value for the transformation. Trans is the point at which the transition is made between the log transform and a linear transform.

The transform statement uses the following form:

MLOG10 shift [lowclamp [transition]]

where:

  • Shift
    Is a real number which is added to the value to be transformed, prior to applying the transform, and which is subtracted from the result of the anti-transform.

  • Low
    Is a real number greater than or equal to zero, to which the shifted value is compared. If the shifted value is less than lowclamp, then lowclamp is the value that is transformed. Otherwise, the shifted value is transformed. Similarly, the anti-transformed value is forced to be no smaller than lowclamp.

  • Trans
    If transition is supplied, then any shifted value that is greater than or equal to transition is transformed normally using the log transformation. However, any shifted value which is less than transition is transformed using a linear transform to a straight line having a slope equal to the first derivative of the log of the value and including the log of the value as a point in that line.

    Note:  Transition must be greater than lowclamp.

Notes and examples

The input to the transform should be a real number which, when shifted (and clamped), is greater than zero. An input of zero or a negative number will cause the transform to fail and will mark that variable or attribute as a bad value.

This example shifts each value up by 1.0 before taking its common log:

MLOG10 1.0

This example clamps each value to a minimum of 0.01 before taking its common log:

MLOG10 0.0 0.01

This example clamps each value to a minimum of 0.001 before transforming; further, any values less than 0.5 are transformed linearly rather than using the common log:

MLOG10 0.0 0.001 0.5

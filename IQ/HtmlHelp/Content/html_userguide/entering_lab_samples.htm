Lab Data Collection (LDC) module

Creating a Lab Data Collection Module

In Aspen Production Control Web Server, after you access the Operations or Engineering view for a particular IQ, you can enter lab sample time and lab sample values if the IQ is configured for Lab Data Collection.

To do this, you must have selected option [3] â€“ "String + Flag" for the sample time option, STIMEOPT, when you configured how new lab sample times would be entered in the IQconfig Lab Data Collection LDC module. Also, all five of the following IQconfig LDC parameters must be configured with keyword LOCAL:

SAMPFLAG

sample taken flag

STKTMSTRIN

sample taken time input

NEWLAB

new lab value

AVLFLAG

new lab value available flag

CNCLFLAG

lab sample canceled flag

To enter a lab sample:

  1. In Aspen Production Control Web Server, access the Operations or Engineering view for a particular IQ. The view provides three panels for PREDICTION, LAB, and ANALYZER information.

  2. In the LAB panel, click the Lab button. The Enter Lab Value dialog box is displayed.

  3. At the top of the dialog, the Current Value of the most recently entered lab value is displayed as read-only.

    At the bottom of the dialog, the Lab Sample History table displays the ten most recent lab samples as read-only.

  4. Use the Enter Sample Time and Enter Lab Value data-entry boxes to enter data for a sample as the information becomes available.

    For example, time information may be available immediately, while sample values (as determined by the lab) may not be available for several hours. You may be in the process of entering several samples at any given moment in time (that is, you may have entered several sample times without entering any values). Aspen IQ will keep track of all this information for you, and you can revisit the dialog to enter sample values at a later time.

    Note:  If either of the data-entry boxes in the Enter Lab Value dialog is gray, or disabled, you can pause the mouse pointer over the data-entry box to see a tooltip explaining the reason the value cannot be entered or revised. For example, the time may be disabled because the entry is in "Waiting" mode, or the data-entry boxes may be gray because the IQ is configured so that lab values are provided by the DCS system, instead of manual entry.

  5. If you enter a sample time without a sample value, the following occurs:

    • The SAMPFLAG and STKTMSTRIN parameters are updated. These parameters are read the next time the application runs on the IQ server. (You will not be able to enter a new sample time until the application runs and processes those values.)

    • After the sample time is entered and processed, the lab sample is in "Waiting" mode. You can enter a lab value, later, when it becomes available.

  6. When you enter a sample value (either after a sample is in "Waiting" mode or simultaneously entered with the sample time), the following occurs:

    • If the lab sample is in "Waiting" mode, it sets the AVLFLAG and NEWLAB parameters, which will be read the next time the application runs on the IQ server. You will not be able to enter another lab value until the current entry is processed, even if there are more sample times outstanding.

    • The average biased prediction at sample time value (LDCPRDST) is the sum of the current PREDBIAS and the average unbiased steady-state or dynamic prediction near the sample time (PLDCPRDST). LDCPRDST is available to the operators as they enter the lab sample value to provide insight as to the direction the lab bias is likely to move once the sample is entered.

  7. Click OK after completing the desired data entries.

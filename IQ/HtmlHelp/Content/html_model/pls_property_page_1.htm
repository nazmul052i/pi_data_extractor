PLS Property Page

The PLS property page shown below appears in the Expert Parameters dialog for the Align Data, Select Variables, Analyze Dynamics, and Build . . . steps:

Note:  Fields in the Fuzzy PLS section are only enabled when the Expert Parameters dialog is activated from the Model dialog (the Build Inferential Sensor step), and the Fuzzy PLS model type is selected.

The General section on the PLS tab allows you to specify parameters for the standard linear PLS algorithm.

  • The Maximum Latent Variables edit box indicates the maximum number of latent variables that the PLS algorithm should consider.

  • The Cross Validation radio button indicates whether Cross Validation should be performed. When Cross Validation is selected, the training dataset is partitioned into the number of subsets specified in the Sets: edit box. Then each subset is consecutively marked for validation (i.e., not used in model training) and a PLS model is built with all the remaining data. The variance that the resulting PLS model captures when it is run with the validation subset is recorded as a function of the number of latent variables in the model. The optimum number of latent variables finally selected is the number that corresponds to the PLS model that maximizes the average of validation variances.

  • If the No Cross Validation radio button is selected, the Threshold: edit box is enabled and you can specify the PLS threshold. This threshold represents the minimum increase in captured variance that a latent variable must contribute in order to be retained in the model.

The Fuzzy PLS section on the PLS tab allows you to specify parameters for the Fuzzy PLS algorithm. In the Fuzzy PLS algorithm, for each latent variable a fuzzy (nonlinear) PLS fits a curve through the variableâ€™s x and y scores (as contrasted with a linear PLS, which fits a straight line through x and y scores). The curve fit is established by first grouping the scores into a number of grid points (as specified in the Grid Points: edit box). The grid points are then distributed either in an equi-distant or equi-probable manner with respect to the x scores axis. Finally, a nonlinear curve comprising a sum of Fit Kernels is constructed through the points, using Fit parameter values as described below:

  • If the Cascaded check box is checked, final latent variables are selected based on a cascaded Fuzzy PLS. A second Fuzzy PLS is built from the scores and residuals of the first Fuzzy PLS; the results are then used for the final curve fit.

  • The Minimum Fit Kernels: edit box allows you to specify the minimum number of Fit Kernels.

  • The Maximum Fit Kernels: edit box allows you to specify the maximum number of Fit Kernels.

  • If the Optimize Fit Parameters check box is checked, relevant Fit parameters will be automatically optimized.

  • If the Optimize Fit Parameters check box is not checked, you can explicitly set Fit optimization parameters:

  • The Grid Points: edit box allows you to specify the number of grid points.

  • The Grid Point Distribution: and Kernel Distribution: drop-down lists (illustrated below for Grid Point Distribution:) allow you to specify how grid points are distributed.

    • Equi-Distant means that grid points are distributed such that they are equally spaced from each other.

    • Equi-Probable means that grid points are distributed such that they are grouped in bins that contain equal quantities of data points.

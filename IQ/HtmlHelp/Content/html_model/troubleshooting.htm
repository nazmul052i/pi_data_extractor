If a pop-up dialog similar to that shown below is displayed when you click the Open button in the Data Specification dialog, you have attempted to load a data file whose name is the same as a data file that is already associated with the current model. If the dataset in fact contains different data, you must rename the file (be sure that you donâ€™t inadvertently rename the original data file). Or, you can delete the file from the list of current data files in the Data Specification dialog, and then click the Open button to re-open the file.

If a pop-up dialog similar to that shown below is displayed when you close the Data Specification dialog, you failed to designate a dependent variable. Click OK to close the error dialog, then open the Data Specification dialog again and designate a tag to be the required dependent variable.

Any time you make a change to a model that would affect later stages of the model building process, a warning dialog similar to the dialogs below will be displayed:

If you click OK, the change will be made and, if needed, boxes representing subsequent model steps will dimmed. If you click Cancel no changes will be applied to the selected block of data.

Note: If you click the Don't Show This Again check box on the dialog, then the warning will be suppressed if you make further changes to the model that might invalidate later steps. You can later restore displaying warning messages through the Options... dialog, as described in The Tools Menu.

Not enough correlations...

If, when you are attempting to build a Sensor Validation model, a message indicating Not enough correlations between variables to build SV component is displayed, there may not be sufficient redundancy in the training dataset to build a good Sensor Validation model (Multivariate Sensor Validation assumes there is redundancy in the data).

You may be able to prevent this error by reducing the Minimum Reconstructability parameter in the Build Expert dialog, which will allow more variables in the model, but may also reduce the reconstruction accuracy for the sensor(s) of interest). It is better to add more physical variables to the model.

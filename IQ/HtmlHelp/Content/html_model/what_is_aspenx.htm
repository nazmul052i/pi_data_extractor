What is Aspen IQmodel?

Aspen IQmodel adds extensive offline empirical modeling capabilities to the Aspen Inferential Qualities (Aspen IQ) product suite. IQmodel is a powerful yet easy-to-use program that supports developing and deploying robust sensor validation and inferential sensing models.

Effective sensor validation requires both identifying faulty sensors and reconstructing faulty sensor values. Traditional approaches to sensor validation emphasize value reconstruction, and use a heuristic identification method based on the error between reconstructed values and measured values. In contrast, the AspenTech approach emphasizes, and is optimized for, faulty sensor identification. If faulty sensors are immediately and accurately identified, reconstructing values is straightforward.

Effective inferential sensor models must function in a variety of manufacturing environments and must handle a wide range of process states and conditions. In addition to providing a control engineer maximum flexibility to incorporate process-specific knowledge through adjustments in model building parameters, Aspen IQmodel offers the following models:

  • Linear PLS (Partial Least Squares) models

  • Non-linear fuzzy PLS models

  • Hybrid (combined) linear PLS and neural network models in which a neural network provides non-linear corrections to the linear model

  • Monotonic neural network models in which the direction of response of the inferred variable to any change in an independent model variable is guaranteed to be consistent at all operating points.

IQmodel runs as a stand-alone Microsoft® Windows™ application.

A dataflow diagram in the main IQmodel Commands window serves as both the primary IQmodel visual interface and a step-by-step guide to building models. Each box in the Commands window represents an action step in the modeling process. By iterating through the steps, a process engineer can readily construct and evaluate multiple models based on the original data source, and then determine which model is best suited for the specific process manufacturing problem.

Radio buttons in the upper left corner of the Commands window identify the modeling mode. When Run Mode is selected, the action steps are related to running a model (offline) using datasets other than the original training dataset.

When building a model, you sequentially click each action box to open a dialog that allows you to specify parameters and options related to the step. As each step is successfully completed and the associated dialog closed, the box for the next step becomes available.

To create a model, plant historical and/or step test data is loaded and conditioned, (optionally) a sensor validation model is built (trained), initial nominal dead time offsets are computed to time-align the data, an optimal set of model input variables is selected, final model dead times and dynamics are analyzed, and the inferential sensor model is built. The performance of either model can be further tested through offline execution using other datasets from the target process. After the models have been fully validated offline, they can be deployed through Aspen APC Manage, an online applications management utility.

Aspen IQmodel is built upon several core technologies. Genetic algorithms (GAs) are used to perform exhaustive searches in the solution space for nominal dead time relationships, model dynamics, and subsets of model input variables. A Linear Partial Least Squares (PLS) technique is used to represent the linear relationships among process variables. Optionally, a Fuzzy (nonlinear) PLS or a hybrid model consisting of a neural network combined with the Linear PLS can be used to construct an extremely robust nonlinear model of a target property value that can be deployed under a wide range of operating conditions. A monotonic neural network model can be used to provide inferential models with guaranteed response direction at all operating points.

During initial data specification for a new model, basic model and data characteristics are identified. This includes whether a sensor validation model is desired (and if so, whether the final model is only for sensor validation).

After plant data files are loaded and conditioned based on user defined parameters, if the model includes the Sensor Validation component, a multivariate sensor validation model is constructed and trained, using the available conditioned data. When a sensor validation model exists, it can be immediately deployed for use online, independent from the associated inferential sensor model.

  • During the data alignment step, IQmodel uses search and fitness parameters provided by the process engineer to govern a genetic algorithm search for optimum static dead time relationships between all independent input variables and the single model dependent output variable (the target property value).

  • In the variable selection step, analogous GA parameters are specified to drive the search for an optimal set of input variables that have the greatest influence on the target property.

  • In the model dynamics step, the GA simultaneously determines first-order process dynamics as well as static dead time relationships based on the final set of variables selected as model inputs.

  • In the model building step, a linear PLS model, a fuzzy PLS model, a hybrid PLS/neural network model type, or a monotonic neural network model type can be specified. The hybrid model uses the linear PLS model scores and outputs to train a neural network for error correction. The result of the model building step is an inferential sensor for the target process property.

After every step, grid (spreadsheet) and/or graphical views of model output provide information that the process engineer can use to obtain greater insight into the nature of the process as well as the solutions (or intermediate results towards solutions) generated by IQmodel.

Model Grid Views

IQmodel grid views display information in a spreadsheet-like format. Grid views of IQmodel data and results are made visible by selecting the desired grid from the drop-down menu located to the right of each box that represents a model building step in the Commands window.

The grid views associated with each model building step are:

  • Specify Data

    • Input Data Grid
    • Input Data Statistics Grid

  • Condition Data

    • Conditioned Data Grid
    • Conditioned Data Statistics Grid
    • Correlations Between Variables Grid

  • Build/Run Sensor Validation

    • Detection Index Grid
    • Sensor Values and Faults Grid

  • Align Data

    • Data Alignment Results Grid

  • Select Variables

    • Variable Selection Results Grid

  • Analyze Dynamics

    • Dead Times / Dynamics Results Grid

  • Build/Run Inferential Sensor

    • Model Statistics grid
    • Model Results grid
    • Model Analysis grid
    • Model Predictions grid
    • Univariate and Sensitivity Analysis grid

  • Sensor Validation Expert Grid Views

    • Structured Residual Transforms (Sensor Values) grid
    • Structured Residual Transforms (Equation Error) grid
    • Structured Residuals grid
    • Equation Errors grid
    • Generalized Likelihood Ratio grid
    • Normalized Cumulative Mean grid
    • Normalized Cumulative Variance grid
    • Reconstruction Model grid

Grid views can also be accessed through the Grid sub-menu that is added to the IQmodel View menu at the time a model is opened:

After a specific grid view is selected, a Grid menu that contains commands for manipulating data in a grid is added to the IQmodel menu bar (and the Commands and Tools menus are removed from the menu bar):

Note:  Only commands that can be executed in the context of the visible grid are available.

Icons in the Toolbar for a grid view also provide single-click short-cuts to the commands available in the Grid menu.

  • Click the Diskette icon to export grid data as a flat ASCII file. The file name and location is specified through a standard Windows file save dialog. Each row in the grid is output as a line in the file. Each line is terminated with a carriage return/linefeed sequence. Each column in a row becomes a field in the corresponding line in the file; fields are delimited by tab characters.

  • Click the Auto-Resize Columns icon to automatically resize column widths to fit displayed data values.

  • Click the Resize Window to Grid icon to automatically resize the grid window to fit the current grid (if the window is wider than necessary to display all grid data).

Note:  The usual Windows techniques for bringing hidden data into view using scroll bars, or resizing windows by dragging corners or edges can be used to select the visible area of a grid.

Grid view Title bars also contain standard Windows control icons.

  • To close a grid, click on its close box.

  • To keep the grid open, but out of the way, by click the minimize box).

  • To enlarge a grid to its maximum size, click the maximize box.

Note:  Grid views are automatically minimized if you open a dialog from the Commands window when a grid is visible.

  3. If you change the status of a block of data, a warning dialog similar to the dialog below will be displayed:

  4. In the Warning message, the following occurs, depending upon your choice:

The Conditioned Data Statistics Grid

The Conditioned Data Statistics grid is accessed through the drop-down menu located to the right of the Condition Data box. Click the Drop Menu button and select Conditioned Data Statistics:

Summary statistics for all tags after the data conditioning process are displayed in the Conditioned Data Statistics grid view.

Note:  Use the horizontal slider to view columns that are not visible.

  • The Status column indicates the status of the corresponding variable. Dependent means the variable is the dependent variable for the Inferential Sensor model. Active means the variable was processed during the Condition Data step and is available for use in subsequent model building steps. Inactive means the variable was not processed during the Condition Data step and is not available for use in subsequent model building steps.

  • The Good Samples column indicates the number of good samples for the corresponding variable.

  • The Bad Samples column indicates the number of bad samples for the corresponding variable.

  • The Cond. Samples column indicates the number of samples whose values were conditioned (based on parameters specified in the Data Conditioning dialog) for the corresponding variable in the row.

  • The Flatline Period column indicates the period (in absolute time units) that the variables values must remain below the Flatline Threshold to generate a Flatline fault. In other words, when a data sample falls below the Flatline Threshold, a timer for the Flatline Period is started, but no fault is generated (assuming Flatline detection is enabled). If all data samples subsequently collected during the Flatline Period remain below the Flatline Threshold, they are also considered good; however, if the first sample collected after the Flatline Period elapses is below the Flatline Threshold, the sample is marked bad and a Flatline fault is generated. Conversely, if before the Flatline Period has elapsed a data sample equals or exceeds the Flatline Threshold, the Flatline Period timer is turned off and no fault is generated.

  • The Flatline Threshold column indicates the threshold value that the variables value must fall below to cause a timer to start for the Flatline Period used to identify Flatline fault conditions (if Flatline detection is enabled).

  • The Spike Period column indicates the period (in absolute time units) that the variables values must remain above the Spike Threshold in order for subsequent data values to be considered normal. In other words, when a data sample exceeds the Spike Threshold, the sample is marked bad, a Spike fault is generated, and the Spike Period timer is started (assuming Spike detection is enabled). If all data samples collected during the Spike period remain above the Spike Threshold, they too are marked bad; however, the first data sample collected after the Spike Period has elapsed is marked good (the process is considered to have moved to another operating level).

  • The Spike Threshold column indicates the threshold value, which, when exceeded on a sample-to-sample basis, triggers a Spike fault (if Spike detection is enabled).

  • The Raw Data Min. column indicates the minimum value in the original raw dataset for the corresponding variable.

  • The Lower Cutoff column indicates the lower limit value (if specified) for the corresponding variable.

  • The Cut Data Min. column indicates the minimum value in the conditioned dataset after limits were applied for the corresponding variable.

  • The Cut Data Avg. column indicates the average value in the conditioned dataset after limits were applied for the corresponding variable.

  • The Cut Data Max. column indicates the maximum value in the conditioned dataset after limits were applied for the corresponding variable.

  • The Upper Cutoff column indicates the upper limit value (if specified) for the corresponding variable.

  • The Raw Data Max. column indicates the raw maximum value in the dataset for the corresponding variable.

  • The Mean Abs. Deviation column indicates the mean absolute deviation in the dataset for the corresponding variable.

  • The Standard Deviation column indicates the standard deviation in the dataset for the corresponding variable.

  • The Variance column indicates the variance in the dataset for the corresponding variable.

  • The Skewness column indicates the skewness (a measure of the symmetry of the distribution) in the dataset for the corresponding variable.

  • The Kurtosis column indicates the kurtosis (a measure of the sharpness in the peak of the distribution) in the dataset for the corresponding variable.

The Data Alignment Results Grid

The Data Alignment Results grid is accessed through the drop-down menu located to the right of the Align Data box. Click the Drop Menu button and select Data Alignment Results as shown below:

The results of the GA search for nominal dead times used to time-align input data will be displayed as illustrated below:

The column on the left side of the grid contains labels for data in each grid row.

  • The first (RSQ) row contains the R2 values for the PLS models built using results from each execution of the genetic algorithm (the execution count is defined through the GA[1] property page in the Expert Parameters dialog for the Align Data step).   • The GA Min column contains the minimum dead time value allowed for the GA search.

  • The GA Max column contains the maximum dead time value allowed for the GA search.

  • The GA Base column contains the base time period used for dead time calculations (i.e., the time resolution of dead time values).

  • The Confidence column contains the confidence associated with the corresponding dead time generated by the best GA run.

The Model Analysis Grid

The Model Analysis grid is accessed through the drop-down menu located to the right of the Build Inferential Sensor box. Click the Drop Menu button and select Model Analysis:

Note:  The Model Analysis grid is only available for the PLS and Hybrid models. It is not available for the Monotonic Neural Network model.

The Model Analysis grid contains the results of PLS model computations for each dependent variable data sample in the training set.

Note:  The picture immediately below illustrates values from the left-most portion of the grid; the next picture illustrates values from the right portion. Use the horizontal slider to select which columns you wish to view (the left-most time stamp column is always visible).

  • The T2 column contains the Hotellings T2 statistic calculated from the independent variable values that were used (after dead time alignment and dynamic filtering) to calculate the dependent variable value at the corresponding sample time. A high T2 statistic indicates that the sample differs considerably from the average, and thus implies high variability within the basic structure of the PLS model.

  • The T2_95 column contains the associated 95% confidence limit.

  • The Q column contains the residual computed for the latent variables at the corresponding sample time. The residual is an indication of how well the sample fits into the PLS model. Larger residuals imply poorer fits.

  • The Q_95 column contains the associated 95% confidence limit.

  • The Leverage column contains values that indicate how much influence the corresponding sample has on the model. Larger numbers indicate more influence.

  • The Stud Res column contains the Studentized Residual statistic for the data sample. The Studentized Residual is an indication of how well the model fits the output variable.

Note:  The Leverage and the Studentized Residual statistics combine to make the best tool for detecting unwanted outliers (samples that have both a large Leverage and a large Studentized Residual).

  • The column labeled LV_X(1) contains the input scores for latent variable 1.

  • The column labeled LV_Y(1) contains the output latent variable score for latent variable 1.

  • The column labeled f(LV_X(1)) contains the PLS model output for latent variable 1.

  • The remaining columns contain score and model output values for the remaining latent variables.

A linear PLS model fits a straight line through the points [LV_X(i), LV_Y(i)], where LV_Y(i) is the linear function of LV_X(i). Similarly, with a nonlinear (fuzzy) PLS model LV_Y(i) would represent a nonlinear function.

The Univariate and Sensitivity Analysis Grid

The Univariate and Sensitivity Analysis grid is accessed through the drop-down menu located to the right of the Build Inferential Sensor box. Click the Drop Menu button and select Univariate and Sensitivity Analysis:

The Univariate and Sensitivity Analysis grid shows how the value of the dependent variable changes with respect to the value of each independent variable. The following grid illustrates the results for a linear model:

This grid is organized into triplets of columns. The first column of each triplet (for example, C4_C2) represents a specific independent variable and contains values for this variable increasing across the full range of the variable (and in fact extrapolating outside of its range). The second column of the triplet (for example f(C4_C2)) shows the corresponding values of the inferred variable, where, for non-linear models, all other independent variables (the "complementary" variables) are held constant. The third column of the triplet (for example f(C4_C2)) shows the corresponding sensitivity (i.e. the derivative) of the inferred variable with respect to the independent variable at that value, where, again, the complementary variables are held constant.

By default, complementary variables are kept constant at the mid-point of their range (their Average Value). If you do not want to use the mid-point value for the complementary variables, then right-click on the grid to display the Evaluations Options& button. Then left-click that button to open the Evaluations Options dialog box.

The Evaluations Options dialog box contains the following options:

  • Values at Mid-points: Average values are used for all complementary variables.

  • Random Values: Random values are generated for all complementary variables.

  • Values from Data Set: Values from a row of the collected data set will be used for the complementary variables. You may select the row of the data set that is used by using the scroll bar. The text box under the scroll bar will display the selected row number.

Note:  Selections made in the Evaluations Options dialog box are immediately reflected in the grid or chart when you click the Apply button.

The value of the independent variable named in the column is varied the range of its values in the training dataset and an extrapolation range of plus or minus 10%. The extrapolation boundaries are shown in red on the Univariate Analysis graph.

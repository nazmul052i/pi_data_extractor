Deploying Models for Use Online

To export a model for use online with real-time process data, click the Deploy button attached to the box that represents the build step for the model you wish to deploy (Build Sensor Validation or Build Inferential Sensor).

Note:  The Deploy button is only available if a valid model exists (the corresponding model must have been built, though its status may be Out of Date). Also, all dialog boxes must be closed (only the Commands window should be visible).

Files used to deploy models contain only the information needed for online monitoring and control (in particular, files do not contain input data, since the model will obtain data from an online source).

When you click the Deploy button for a sensor validation model, the following dialog appears. The default directory is the directory set for Runtime files in the Options: dialog. Sensor validation models automatically are given the extension .svr.

When you click the Deploy button for an inferential sensor model, the following dialog appears. The default directory is the directory set for Runtime files in the Options: dialog. Inferential sensor models automatically are given the extension .iqr.

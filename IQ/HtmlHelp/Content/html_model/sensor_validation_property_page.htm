Sensor Validation Property Page

The Sensor Validation property page appears in the Expert Parameters dialog for the Build / Run Sensor Validation step:

The Build Parameters section contains parameters that are set when a model is built and cannot be modified online.

  • The Data Smoothing Factor is the coefficient of a first order moving average filter that is applied to the available data before the sensor validation model is built. The Data Smoothing Factor must be greater than or equal to zero, and less than 1.0. A value of 0.0 means no smoothing. A value very close to 1.0 means substantial smoothing. The Data Smoothing Factor is established when a model is built; the Data Smoothing Factor can not be changed when a model is subsequently run online.

  • The Minimum Reconstructability parameter is a threshold value that, when a fault condition is detected during model training, serves as a measure of how much better the value computed by the sensor validation model is than the average value for the variable. Variables (tags) whose Reconstructability falls below the Minimum Reconstructability value will not be incorporated in the sensor validation model (their values will not be replaced by values calculated by the sensor validation model in the event of sensor failure). Reconstructability is based on the variance of the reconstruction error when compared with the variance that results from using average values rather than the model estimate. If the Minimum Reconstructability parameter is 0.0, no faulty values will be replaced by sensor validation model values.

The Run Parameters section contains parameters that are established when a model is built, but which also can be subsequently modified online if operating conditions warrant.

  • The Index Smoothing Factor is the coefficient of a first order moving average filter, which is used to smooth the Filtered Fault Detection Index (FFDI), and the Normalized Squared Filtered Structured Residuals (NSFSR). The Index Smoothing Factor must be greater than or equal to zero, and less than 1.0. A value of 0.0 means no smoothing. A value very close to 1.0 means substantial smoothing. An initial value for the Index Smoothing Factor is established when a model is built; however, the Index Smoothing Factor can be changed when a model is deployed online if operating conditions warrant.

  • The Detection / Identification check box indicates whether the sensor validation fault detection and identification mechanisms are active. When the box is checked, the mechanisms are active.

  • The Detection Confidence Percentage is used to set up the confidence intervals used for fault detection. The percentage value must lie between 90 and 100, inclusive.

  • The Identification Confidence Percentage is used to set up the confidence intervals used for fault identification. The percentage value must lie between 90 and 100, inclusive.

  • The Identification History Size edit box specifies the size of the history buffer that the identification logic maintains. The history buffer can be between 1 and 200 samples, inclusive.

  • The Classification check box indicates whether the sensor validation fault classification mechanism is active. When the box is checked, the fault classification is active.

  • The Classification History Size edit box specifies the size of the history buffer that the classification logic maintains. The history buffer can be between 1 and 200 samples, inclusive.

  • The Replacement check box indicates whether the sensor validation replacement mechanism is active. When the box is checked, the sensor validation model will replace faulty sensor data (for variables whose reconstructability is above the Minimum Reconstructability) with estimated values.

The Model Dialog

The Model dialog is used to specify a model type and build an inferential sensor model. Through the associated Expert Parameters dialog additional parameter settings for the selected model type can be specified:

The Variables section contains a summary of information about model independent variables.

  • The Name column indicates each independent variable’s tag name.

  • The Status column indicates the status of each variable. Active means the variable is used in the current model. Off means the variable is not used.

  • The Dead Time column contains the dead time for the variable.

  • The Filter column contains the dynamic filter value for the variable.

  • The Monotonicity column appears only when the Model Type: Monotonic Neural Network is selected. This column contains the desired monotonicity for the variable. After you build the model, the column will change any Unknown monotonicities to the value calculated during training.

  • The Model Status: edit box indicates the current model status at this step (Invalid, Ready, or Out of Date, as previously defined).

  • The Variables: edit box indicates the number of model input variables.

  • The Base Time: edit box indicates the base sampling interval for the model.

The Results Section

The Results section displays information about model output (after a model has been built). When the Build dialog is first opened for a given model, all values in the Results section are 0.

  • The Set column indicates which dataset the labeled row corresponds to:

    • The Trn row contains model results using input data reserved for training.

    • The Val row contains model results using input data reserved for validation.

    • The All row contains model results using the entire input dataset.

  • The Samples column contains the number of samples for the corresponding model execution.

  • The RSQ column contains the R-Squared statistic of the results for the corresponding model execution.

  • The RMS Error column contains the Root Mean Square error in results for the corresponding model execution.

  • The Avg Error column indicates the average error in results for the corresponding model execution.

  • The Max Error column indicates the maximum error in results for the corresponding model execution.

  • The Correlation column indicates the correlation in results for the corresponding model execution.

  • The Dep: box contains the tag name of the model’s dependent variable.

  • The Model: box indicates the type of model that generated the results shown.

  • The LV: box contains the number of latent variables in the model.

  • The Hidden Units: box contains the number of hidden units in the neural network component of the model if the model type is Hybrid: PLS – Neural Network or Monotonic Neural Network. In the former case, the number of hidden nodes is selected by the neural net training algorithm, so this text box provides the results of this automatic selection.

    Note:  The value is 0 for Linear and Fuzzy PLS models.

The Model Section

The Model section contains a drop-down list that allows you to specify a model type:

  • The Linear PLS type indicates that the model will be the standard linear PLS.

  • The Fuzzy Nonlinear PLS type indicates that the model will be a nonlinear PLS model.

  • The Hybrid: PLS – Neural Network type indicates that the model will be a linear PLS combined with a neural network to track errors between the linear PLS and input data.

  • The Monotonic Neural Network type provides inferential models with guaranteed monotonicity. This means that the direction of response of the inferred variable to any change in an independent model variable is guaranteed to be consistent at all operating points. Four options are available for each independent variable:

    • Positive Monotonic: enforces the condition that, no matter what operating point the system is at, increasing the specified independent variable is guaranteed to lead to an increase in the inferred variable.

    • Negative Monotonic: enforces the condition that, no matter what operating point the system is at, increasing the specified independent variable is guaranteed to lead to a decrease in the inferred variable.

    • Unknown Monotonic: enforces a monotonic condition, but indicates that the monotonic direction (positive or negative) is unknown. The system will identify the monotonic direction during training. After training, the calculated values will be displayed in the Monotonicity column in the Model dialog.

    • Non-Monotonic: indicates that No monotonic condition is enforced for that particular variable. The purpose of this is to allow a mixed model in which some variables have a monotonic relationship with the inferrred variable and some do not.

The Model Properties Dialog

The Model Properties dialog allows you to specify whether or not to use the associated input variable(s) in the building the final model, and allows you to explicitly change model dead time and dynamics values.

Note:  The names of all variables that were highlighted when you opened the Properties: dialog are displayed in the title bar of the dialog.

Caution:  The fields in the Model Properties dialog will display the properties of the topmost variable in the selected list. Unless the changes you want to make to all the selected variables are identical, do not select several variables at the same time.

The Status section allows you to specify whether a variable should be used in the model.

  • If the Active (On) radio button is selected the variable will be used in the model.

  • If the Inactive (Off) radio button is selected the variable will not be used in the model.

    Note:  To quickly turn all inactive variables back on, select Restore from the pop-up menu shown above. If you want to individually turn variables back on, highlight the variable whose state you want to change and click Properties… to open the Properties: dialog.

The Dead Time section allows you to directly specify the dead time for the variable(s).

The Dynamics section allows you to directly specify the dynamic filter value for the variable(s).

The Monotonicity combo list allows you to select the monotonicity prior to building the monotonic neural network model. This list will only be active if you have selected the Monotonic Neural Network model type. The selection will be initialized to the current monotonicity value of the selected variable (or the first variable if several variables are selected.)

The Build Button

Clicking the Build button will generate the model. As the model is trained, a dialog box indicates progress. If you selected the Monotonic Neural Network model type, you can observe the training progress through the standard status dialog or through a predicted versus observed chart.

The following messages can appear on the status dialog:

  • Current Iteration out of Maximum Iterations: Indicates the number of iterations the training process has taken.

  • Current training error: Indicates the difference between inferred data and lab data.

  • Best training error: Indicates the smallest difference (closest match) between inferred data and lab data.

  • Current validation error: Indicates the difference between inferred data and lab data

  • Best validation error: Indicates the smallest (closest match) difference between inferred data and lab data.

Note:  You can use the Cancel button to halt the training. In this case, the current best weights will be used.

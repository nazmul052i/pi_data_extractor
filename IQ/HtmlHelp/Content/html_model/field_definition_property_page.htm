Field Definition Property Page

The Field Definition property page is part of the Expert Parameters dialog that is accessed through the Expert button on the File Format dialog (The File Format dialog is displayed when a data file is first loaded through the Data Specification dialog or when the Format... button on the Data Specification dialog is clicked). The Field Definition property page is used to define characters that separate fields in records:

The Location: drop-down list contains two entries:

The content of other fields applies to the current selection in the Location: drop-down list.

  • The Main Header record type indicates that delimiter definitions apply to file records in the Main Header of a file, as specified on the main File Format dialog.

  • The Data record type indicates that delimiter definitions apply to file data records.

  • In the Delimiters section, the Values: view contains check boxes that indicate the characters that are available as delimiters for the record type selected in the drop-down list.

  • The Values: view contains check boxes that indicate the characters that are available as delimiters for the record type selected in the drop-down list. When an item is checked, the corresponding character will be interpreted as a field delimiter for the selected record type. The check box with no label is a place holder for a custom delimiter.

    For example, in the picture above, there are 4 characters available to serve as delimiters: a Tab, a Space, a Comma, and a Semi-Colon; however, only the Tab will be interpreted as a delimiter during input file parsing.

  • The Custom: edit box is used to identify additional characters that should be interpreted as delimiters for the currently selected record type.

    Note:  Although multiple characters are accepted by the Custom: edit box, only the first character is used.

    • To add a character, simply enter it into the Custom: edit box. The character is immediately displayed in the Values: view (a check box to enable/disable interpreting the character during parsing is automatically placed in the view).

    • To remove a character from the list, select it in the Values: view and then press the Delete or Backspace key (the four IQmodel standard delimiters shown above cannot be removed).

The Fixed Fields section is used to define the location and size (width) of fixed size fields:

Note:  Fixed fields must be sequential, and the first fixed field must be the first field in a record.

The Number of Leading Fixed Width Fields: edit box is used to specify the number of fixed fields. When you enter a non-zero number, fields labeled Field1 through FieldN (where N is the number you entered) are placed in the Fixed Fields Widths: list view as illustrated below for 2 fields:

The default field width is 8 characters. Field start positions are automatically computed based on the widths of preceding fields.

To change the width of a field:

  1. Right click on its name, as illustrated below:

  2. Enter the new field width in the Set Fixed Field Widths pop-up dialog:

  3. Click OK to close the Set Fixed Field Widths dialog. The width of the selected field will change, and the start position of subsequent fields will be updated in the Fixed Fields Widths: view as shown below:

    When you click OK to close the Expert Parameters dialog, all the delimiter characters in the Values: list will be associated with the format identified in the main File Format dialog. However, only the characters that are checked will be interpreted as delimiters when the format is used to parse an input file.

    Note:  If you click Cancel to close the Set Fixed Field Widths dialog, the width of the selected field will not be changed.

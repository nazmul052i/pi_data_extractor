IQmodel Reference

The hints and tips presented in this section assume that you have completed the Tutorial and have a solid grasp of the mechanics of building models using IQmodel. Keep in mind that ultimately the validity of any model depends on the quality of the data used to build it, as well as on the experience and judgment of the process engineer who evaluates model output. The key to building optimal models is Einstein's admonition that "Everything should be made as simple as possible, but not simpler."

In general, fewer input tags and fewer variables will result in a model that utilizes fewer computation resources (and thus will offer a more responsive user interface) when deployed online. However, if too few (or inappropriate) inputs are used to build the model, important process characteristics may not be adequately represented. Similarly, if too few latent variables are used, process problems can go undetected because the resulting model does not capture all relevant correlations inherent in the process.

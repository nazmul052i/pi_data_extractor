Overview of Running Models Offline

To run a model with data other than the original training dataset, load the model and click the Run Mode radio button in the Commands window. When loading a model for Run Mode, you can choose one of the following type of files:

  • .iqm file
    This is the default file and it corresponds to a model that you have saved from IQmodel using the File/Save… or File/Save As… menu commands. This type of file maintains all the information about how the inferential model was built. It can be executed either in the Build Mode or in Run Mode.

  • .svr file
    This type of file corresponds to a sensor validation model that you have exported by means of the Deploy button attached to the Build Sensor Validation modeling block. When a .svr file is loaded, only the Run Mode button is available.

  • .iqr file
    This type of file corresponds to an inferential model that you have exported by means of the Deploy button attached to the Build Inferential Sensor modeling block. When a .iqr file is loaded, only the Run Mode button is available.

When IQmodel is in Run Mode, only the Specify Data and Run . . . boxes are available. The availability of Run . . . boxes depends on whether the corresponding model is built, and whether data files are loaded.

The Run Mode Data Specification dialog allows you to specify data files to use and to view the status of all variables in the models that have been built:

The Data Files section contains information about files currently available for use by IQmodel.

  • The Name column contains the names of variables found in the corresponding file(s).

  • The File column indicates the name(s) of the file(s) currently loaded.

To add a data file to the list of files available for input data:

  1. Click the Open... button. The Open... button activates a standard Windows file open dialog:

  2. Locate the file to add to the list of input data files, then click its name to select it.

  3. Click the Open button to open the file. The input data file dialog will close and the File Format dialog will automatically open.

  4. Specify the file format characteristics as detailed in Specifying Input File Format Information.

  5. Click OK to close the File Format dialog. The selected file name will be added to the list of available files visible in the Data Files view, and information about variables in the file will be displayed in the Variables section.

    Note:  The Clear All button removes all files from the list of data files and, as a result, also removes all input data. IQmodel does not permit loading data from multiple files that have the same name.

    The Variables section contains information about model variables.

    • The Name column contains the names of variables used in the current model.

    • The Status column indicates the status of the variable (Dep, Ind, or Off).

    • The Inferential column indicates whether the variable is used in the inferential sensor model.

    • The Sensor Val. column indicates whether the variable is used in the sensor validation model.

    • The Present column indicates whether data for the variable is contained in a loaded file (Yes or No).

  6. Click the Load button after data files are opened to load data into IQmodel. The Data Specification dialog will remain open to permit viewing the status of variables and data in the loaded file(s).

  7. Click the OK button to load data into IQmodel and close the Data Specification dialog.

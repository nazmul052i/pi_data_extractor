The Data Alignment Dialog

The Data Alignment dialog is used to specify parameters that determine how nominal dead times are computed during the input data time alignment process. If a genetic algorithm is used to compute nominal dead times, through an associated Expert Parameters dialog you can further refine GA search parameters.

Note:  The Data Alignment dialog (i.e., the Align Data box) is only available if the corresponding check box was checked (in the Model Defaults dialog) at the beginning of the model building process.

Note: Refer to The PLS Property Page, The GA [1] Property Page, and The GA [2] Property Page in the Expert Parameters Dialogs section for information about setting Expert parameter values.

The Variables section contains a summary of dead time information for each independent variable, reflecting the results of the best genetic algorithm run and/or user-specified values.

  • The Name column indicates each independent variable’s tag name.

  • The Group column contains the variable’s group name if it is a member of a group (if the column is blank, the variable is not a member of a group).

  • The User Set column contains an explicit dead time value (if the column is blank, the variable’s dead time is determined by IQmodel’s genetic algorithm search).

  • The GA Min and GA Max columns contain the current minimum and maximum dead time value limits for the GA dead time search.

  • The GA Base column indicates the time resolution of the GA search (i.e., dead times computed by the GA will be multiples of the GA Base time).

  • The Result column contains the corresponding variable’s dead time (after the GA search has been performed).

On the Statistics: line:

  • The Variables edit box contains the total number of independent variables.

  • The "Free" Variables edit box contains the number of free, or effective variables (i.e., the number of variables that the GA will use for determining dead times).

  • The Groups edit box contains the number of groups currently defined.

The Model Status: edit box indicates the current model status at this step (Invalid, Ready, Out of Date).

  • Invalid means that the operations associated with this step have not yet been performed.

  • Ready means that the operations associated with this step have been completed and results of this step are ready for use in subsequent model steps.

  • Out of Date means that operations associated with this step have been performed and results are available for use in subsequent steps; however, one or more parameters associated with this step have been changed since the operations were performed.

The Base Time: edit box in the Variables section indicates the base sampling interval for the model.

  • The base time is formatted as Days Hours:Minutes:Seconds.

Deleting and Restoring Variables for Data Alignment Processing

To delete a variable (i.e., mark it so that IQmodel does not compute a dead time for it), select Delete from the pop-up menu.

Note:  Deleting a variable does not remove it or its associated data from the underlying model definition file; it simply marks it so that it is not processed for dead times or used in subsequent model steps.

To restore all variables that have been deleted in this model building step, select Restore from the pop-up menu.

Note:  Even though you may delete individual variables, previously deleted variables are restored only as a group. To restore only some of the deleted variables, you must restore them all and then re-delete those you actually want deleted.

To change the parameters that govern the GA dead time computations, select Properties ... from the pop-up menu to access the Data Alignment Properties dialog.

The Data Alignment Properties Dialog

The Data Alignment Properties dialog, shown below, allows you to change parameters that govern how nominal dead time relationships are computed:

The Variables section on the Dead Times tab lists the name(s) highlighted when you clicked Properties on the pop-up menu.

  • The Name column indicates the variables whose properties can be set using this page.

  • The Group column contains the variable’s group name if the variable is a member of a Group (if the column is blank, the variable is not a member of a group).

The Dead Times section allows you to specify values for the dead time parameters that are shown in the main Data Alignment dialog.

Note:  Dead Times are specified as times in the form of Hours:Minutes:Seconds (HH:MM:SS). Zero fields are not displayed. For example, :06 means 6 seconds, and :01:30 means 1 minute, 30 seconds. However, dead time values must be multiples of the Dead Time Base interval. If you use the edit box spinners to adjust dead times, the spinners automatically use increments of the Dead Time Base interval. If you enter a value directly into an edit box, the value is rounded to the closest multiple of the Dead Time Base interval.

  • If the User Set radio button is selected, the adjacent Dead Time: edit box is enabled, and you can explicitly set a dead time value for the variables named in the Variables section.

  • If the GA Search radio button is selected, the adjacent Minimum Dead Time:, Maximum Dead Time:, and Dead Time Base: edit boxes are enabled, and you can enter values for these parameters.

  • The Minimum Dead Time field specifies the lower bound for dead times computed by the genetic algorithm.

  • The Maximum Dead Time field specifies the upper bound for dead times computed by the genetic algorithm.

  • The Dead Time Base: field specifies the basic Dead Time interval (in effect, this is the base sampling interval of input data).

Creating and Deleting Groups

If several input variables are known to have similar time delay characteristics with respect to their effects on the dependent variable, they can be combined into a Group for the purpose of computing nominal dead times. All variables in a group are treated as one variable during the genetic algorithm search for nominal dead times.

To create a group:

  1. Highlight the names of variables you want to include in the group, as illustrated below:

    Note:  If the names are not contiguous, you can control-click to select multiple non-contiguous variables.

  2. Right-click on a highlighted name and select Group... from the pop-up menu.

    The Group: dialog, shown below, allows you assign a name to the new group:

    • The list view in the Variables section lists the names highlighted when you clicked Group... on the pop-up menu.

    • The Name: list box in the Group section is used to enter a name for the new group. The drop-down list contains the names of all currently defined groups.

  3. Click OK to close the Group: dialog and complete the definition.

To delete a group when the main Data Alignment dialog is visible:

  1. Highlight any variable name that is in the group you want to delete.

  2. Right-click to activate the pop-up menu, and select Group... to access the Group: dialog.

  3. Locate the name of the group you want to delete in the Name: list box.

  4. Delete the name, using the Delete or Backspace keys to erase the name from the edit portion of the Name: list box.

  5. Click OK to close the Group: dialog.

The Data Specification Dialog

The Data Specification dialog is used to establish the link between plant data and variables in IQmodel.

Note:  The basic appearance of the Data Specification dialog depends on whether IQmodel is in Build Mode or Run Mode. For information about the Data Specification dialog when IQmodel is in Run Mode, please refer to Running Models Offline.

The Data Files section contains information about data files currently loaded in IQmodel.

  • The File column contains file names.

  • The Variables column indicates the number of variables (tags) represented in the corresponding file.

  • The Records column indicates the number of records in the corresponding file.

  • Click the Open or Format buttons in the illustration above to jump to IQmodel Reference topics addressing the Open and Format functions.

The Variables section contains information about variables in the file(s) listed in the Data Files section:

  • The Name column indicates the names of tags represented in the listed file(s).

  • The Status column indicates whether the variable is an independent variable (Ind), the (single) dependent variable (Dep), or Off (not considered for processing in this and subsequent steps). To change the status of a variable, right click on its name to access the pop-up menu illustrated below, then select the appropriate menu option:

Note:  If a dependent variable is already specified, and you then designate another variable to be the dependent variable, the status of the original dependent variable will change to Off (meaning that it will not be considered when the model is built). If you wish to use the original variable as an independent variable in the model, you must explicitly set its status to Independent.

  • The File column contains the name of the file that contains the raw data for the corresponding variable.

  • The Ind: edit box contains the current number of independent variables.

  • The Off: edit box contains the number of variables currently off.

  • The Dep: edit box contains the name of the single variable currently designated as the dependent variable.

  • The Type: drop-down list allows you to indicate the type of sample data that the dependent variable represents:

    • Continuous indicates that each data record contains a sample value for the dependent variable. All available values are loaded into the model.

    • Laboratory indicates that the dependent variable represents laboratory data sampled at irregular times (though each sample is synchronized with a time when samples are available for all independent variables). Typically, laboratory data represents an average value over some period of time. All available values are loaded into the model.

    • Analyzer indicates that the dependent variable represents data from a hardware analyzer. Every data record contains a value for the dependent variable, but only samples whose value differs from the immediately prior sample are loaded in the model.

The Samples section displays information about samples in data files currently loaded into IQmodel (when the Data Specification dialog first opens, all values are 0).

  • The Base Time: edit box indicates the basic data sampling interval that will be used in all subsequent IQmodel calculations. Initially, IQmodel sets the Base Time to be the smallest Mode Interval (i.e., the most frequently occurring time difference between two consecutive data samples) of all loaded data files. The Base Time value can be modified by clicking the spinners in the edit box. Refer to Input Data Source Properties for additional information.

  • The Independent: edit box indicates the total number of raw independent variable data samples.

  • The Dependent: edit box indicates the total number of dependent variable data samples.

Working with Data Sources

IQmodel is designed to handle input data from text files, as follow:

  • Flat ASCII text files with records terminated by a newline (linefeed or carriage-return or carriage-return / linefeed combination), including formatted DMCplus ASCII clc files.

Note:  For detailed information about clc files, please refer to DMCplus documentation.

Specifying an Input Data File (The Open... Button)

To add a data file to the list of files available for input data:

  1. Click the Open... button. The Open... button activates a standard Windows file open dialog.

  2. Locate the file you want to add to the list of input data files.

  3. Click on the name of the file to select it.

  4. Click the Open button to open the file (the input data file dialog will close and the File Format dialog will automatically open). The file you selected will be added to the list of available files visible in the Data Files view.

Specifying Input File Format Information (The Format... Button)

The File Format dialog is activated by the Format... button on the Data Specification dialog. It is also automatically activated when an input data file is loaded.

The File Format dialog is used to specify the format of data in an input file and to specify how the file should be parsed. The resulting format specifications can be assigned a name and saved for future use with other data files that have the same format.

Note:  The File Format dialog can be opened by the Format... button in order to permit defining and saving format specifications even when no data file is currently loaded.

The Format Section

The Format section contains general information about the current format specifications:

  • The name visible in the Name: drop-down list contains the name of the current format. txtFileN is the name assigned when a new file is loaded (where N is incremented by 1 for each new file).

    To apply another named format to the current file, locate the name in the drop-down list and select it.

  • The File Extension: edit box indicates the file extension that is associated with the current format. (File extensions do not have to be unique. For example, the standard txt extension for text files can be associated with any number of named formats.)

  • The Comment: edit box can be used to enter a brief description of the format to assist other users in deciding whether to use the format with a newly loaded file.

The Header Section

The Header section contains information about header records:

If the Main Header check box is checked, the file contains header information and the corresponding Number of Rows: list box is enabled to allow specifying the number of header rows.

The File Section

The File section contains information about file data:

Note:  If the File Format dialog was opened through the Format... button, the File section contains no data, and fields in it are disabled. However, all other format specification fields are accessible and formats can be defined and saved for future use.

The File: drop-down list indicates the name of the file that contains the data is displayed in the view below the edit box. If multiple files are associated with the current model, their names are in the list. The contents of another file can be displayed by selecting its name from the list.

The Tag Information Section

The Tag Information section indicates where Tag name and unit fields are located:

  • If the Tag Name is File Name check box is checked, the file contains data for only one Tag, and the name of the Tag is the name of the file. This setting implies that multiple files will be loaded.

  • If the Tag Names check box is checked, the corresponding in Row: and Starting Field: list boxes are available to allow specifying the row and the field within a row for Tag names.

  • If the Tag Units check box is checked, the corresponding in Row: and Starting Field: list boxes are available to allow specifying the row and the field within a row for Tag units.

Note:  IQmodel supports a wide range of formats for tag names. Any ASCII character is legal in a tag name except:

  • Carriage Return

  • Line Feed

  • Space

  • ASCII Control Characters

  • Any character currently defined as a field delimiter by the user

The Date / Time Section

The Date/Time section indicates where date and time fields are located:

Note:  Date and time formats are specified on the Date / Time property page in the Expert Parameters dialog associated with the File Format dialog.

  • If the No Date/Time radio button is selected, the file contains no date and time information.

  • If the Date/Time in First Field(s) radio button is selected, the file contains date and time information in the first field(s) of each record.

  • If the Explicit Date/Time in radio button is selected, the file contains date and time information in the field(s) other than the first in each record, and the corresponding Start Date/Time in Row:, Sampling Interval in Row:, and Field: list boxes are made available to allow specifying the row and the field within a row for Date/Time and Sampling Interval information.

Note:  The sampling interval value in the defined field must be an integral number of seconds, not a formatted time.

Save Format, New Format, and Delete Format buttons in the bottom section of the File Format dialog are used to manage file format specifications:

  • Click the Save Format button to save the current format specifications under the name currently visible in the Name: drop-down list in the Format section:

  • Click the New Format button to give the current set of format specifications a new name. IQmodel assigns a default name and description as illustrated below; change the Name: and/or File Extension: and/or Comment: edit boxes to reflect new format characteristics. Then click the Save Format button to save the format under the new name.

  • Click the Delete Format button to delete the current format specifications whose name is currently visible in the Name: drop-down list in the Format section.

Note:  Ensure that the name you want to delete is visible in the edit portion of the drop-down list before you click the Delete Format button.

Deleting an Input Data Source

To remove a file from the list of input files:

  1. Click on a file name in the Data Files list view.

  2. Right-click the file name to activate the pop-up menu illustrated below:

  3. Select Delete from the pop-up menu. The file that was highlighted when you right-clicked will be immediately removed from the list of input data files (and tag data contained in the file will no longer be available to IQmodel).

Input Data Source Properties

To view basic information about an input file, select a file name in the Data Files list view, then right-click to activate the pop-up menu illustrated below, and finally choose Properties... from the pop-up menu.

The Properties: dialog will be displayed.

The File section shows the data file whose name was highlighted when the Properties: dialog opened.

The Variables section contains information about variables represented in the file.

  • The Name list view contains the names of all variables (tags) that have data in the file.

  • The Count: edit box shows the number of variables.

The Samples section contains information about data samples in the file.

  • The Records: edit box shows the number of raw data records.

  • The Minimum Interval: edit box shows the minimum sampling interval (i.e., the minimum time between two consecutive records) for the file.

  • The Mode Interval: edit box shows the most frequently occurring sampling interval (i.e., the most frequently occurring time between two consecutive records) for the file. IQmodel displays the smallest Mode Interval of all loaded files as the suggested Base Time in the Data Specification dialog.

  • The Start Time: edit box shows the date and time of the earliest (first) record in the file.

  • The End Time: edit box shows the date and time of the latest (last) record in the file.

Loading Input Data

  1. Click the OK button in the main Data Specification dialog to accept the current input file specifications and data values, and to close the Data Specification dialog.

  2. Click the Load button to accept the current input file specifications and to load data from the file(s). The Data Specification dialog will remain open so that resulting statistics can be viewed (the caption on the OK button will change to Close).

  3. Click the Cancel button to close the dialog without loading data (the Cancel button will not be available if at least one file has already been loaded).

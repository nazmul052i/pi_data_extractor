Modifying Existing Models

To open an existing model:

  1. Click Open on the File menu, or click File Open in the main window Toolbar.

  2. Locate the model file you want to open in the standard Windows file dialog that appears. After you have selected a model file, the IQmodel Commands window will be displayed. The boxes that are available represent model steps that are already completed.

  3. Click on any available box to open the corresponding dialog and make desired changes. If a change will invalidate data or affect results of later steps, the boxes corresponding to those steps will become unavailable when you make (and accept, by clicking OK or Apply) changes in the dialog youâ€™re working with.

    Note:  Depending on the Warning Messages options that are active, appropriate warning messages are displayed when you make changes to a model. See The Tools Menu for additional information.

About model modifications

Assuming the loaded model has been built, there are essentially three major model modifications that can be used to potentially improve model performance:

  • New data for the same set of input tags can be acquired and used to retrain the model. This modification is particularly relevant if the entire process is operating at a level different from the level that generated the original training dataset.

  • Additional input variables can be incorporated in the model, or tags used in the original model can be made inactive. Changing the number of model inputs can significantly affect the correlation structure of the resulting model.

  • You can change the type of model selected in the Model dialog. Again, this can have significant effects on the behavior of the resulting model, particularly if the underlying target process is highly nonlinear.

In the context of these major changes, there are other changes that can be made to individual model elements (for example, input variables). In general, these other changes are made through property pages.

Access property pages by right-clicking a name in a list view. Property pages are associated with variables in list views of the following model step dialogs:

  • Data Specification

  • Data Conditioning

  • Sensor Validation

  • Data Alignment

  • Variable Selection

  • Model Dynamics

  • Model

Note:  If more than one variable in a view is highlighted when you right-click, any changes you make on the associated property page will be applied to all the variables. Or, if the variable is a member of a group, changes will apply to all members of a group.

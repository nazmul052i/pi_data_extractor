Model Graph Views

IQmodel graph views display information in graphical formats. Graphical views of IQmodel data and results are made visible by selecting the desired graph from the drop-down menu located to the right of each box that represents a model building step in the Commands window.

The graph views associated with each model building step are:

  • Specify Data

    • Input Data Graph

  • Condition Data

    • Conditioned Data Graph
    • Correlations Between Variables Graph

  • Build/Run Sensor Validation

    • Detection Index Graph
    • Sensor Values Graph

  • Align Data

    • RSQ After Data Alignment Graph

  • Build/Run Inferential Sensor

    • Model Predictions Graph
    • Model Analysis graph
    • Univariate Analysis graph
    • Sensitivity Analysis graph
    • Sensitivity vs Rank graph

  • Sensor Validation Expert Graph Views

    • Structured Residuals graph
    • Equation Errors graph
    • Generalized Likelihood Ratio graph
    • Normalized Cumulative Mean graph
    • Normalized Cumulative Variance graph
    • Estimated Fault Size graph
    • Reconstruction Model graph

Graph views can also be accessed through the Graph sub-menu that is added to the IQmodel View menu at the time a model is opened:

After a specific graph is selected, a Graph menu that contains commands for manipulating data in a graph is added to the IQmodel menu bar (and the Commands and Tools menus are removed from the menu bar):

Note:  Only commands that can be executed in the context of the visible graph are available in the Graph menu.

In addition, icons in the Toolbar of a visible graph offer single-click short-cuts to commands in the Graph menu.

The Graph Legend

All graphs share a common legend format, illustrated below for the Input Data graph:

The Tags column lists the names of Tags (variables) whose values can be displayed in the visible graph. To display the values for a Tag, click its name in the legend.

Note:  In the legend for graphs that show computed statistics, the Tags column contains a descriptive name for the statistic being displayed.

If the visible graph is an Overlaid graph, the data for a particular Tag can be selectively shown or hidden by right-clicking the Tag name in the legend and then choosing the appropriate visibility option:

Note:  A line is drawn through the name of each Tag whose data values are hidden.

  • The Status column indicates the status of the corresponding variable. For Tags, the status is either Independent or Dependent, depending on whether the Tag is an independent variable or the dependent variable. For computed statistics, the status is Variables Statistic.

  • The Samples column indicates the number of data samples available for the corresponding Tag. If the variable is a statistic, the value in the Samples column is zero.

  • The Min and Max columns contain, respectively, the minimum and maximum values of the data plotted for the corresponding Tag (or statistic).

For each IQmodel graph, visible attributes of the entire graph can be specified on a graph Properties: page, as described in Graph Properties. Visible attributes for individual tag values shown on a graph can be specified on a tag Properties: page, as described in Tag Properties.

Selecting an Area in a Graph

The Selection command activates a submenu containing graph area selection commands. To choose a selection method, click the corresponding name in the submenu:

A check mark beside a name indicates the corresponding area selection method is active. Only one selection method can be active at a time. If you click a name that has a check mark beside it, the corresponding area selection method is turned off.

  • The Horizontally command allows you to select a horizontal area (i.e., from one point on the x axis to another point on the x axis).

  • The Vertically command allows you to select a vertical area (i.e., from one point on the vertical axis to another point on the vertical axis).

  • The Both command allows you to select an arbitrary rectangular area in the graph (by identifying one corner and dragging diagonally to the opposite corner).

Or, in the Toolbar of a visible Graph:

  • Click the Horizontal Selection icon to select an area from one point on the horizontal axis to a second point on the horizontal axis.

  • Click the Vertical Selection icon to select an area from one point on the vertical axis to a second point on the vertical axis.

  • Click the Both Directions Selection icon to select a rectangular area by dragging from one point to another (the second point identifies the corner of the selected rectangle located diagonally from the first point).

After an area is selected, it can be enlarged using the Zoom In command, or the selected area can be used to identify bad data.

  • To zoom in on an area in a graph, right-click on the area and select Zoom In from the pop-up menu:

  • To restore normal magnification, right-click in the area and select Zoom Out:

For detailed information about using area selection tools to identify bad data and the Show in Grid command, refer to Using the Input Data Graph to Identify Bad Data.

Specifying the Graph Mode

The Mode command activates a submenu containing graph mode commands. To choose a mode option, click the corresponding name in the submenu:

A check mark beside a name indicates that the corresponding graph mode is active. One graph mode is always active.

  • The Single command generates a single graph that plots data for a single tag.

  • The Overlaid command plots data for multiple tags on a single graph. Right-clicking its tag name in the legend area, and then selecting Hide from the pop-up menu can hide a line that represents data for an individual tag.

  • The Stacked command generates multiple graphs, stacked vertically. Each graph plots data for one tag.

Or, in the Toolbar of a visible Graph:

  • Click the Single icon to plot data for a single tag.

  • Click the Overlaid icon to plot data for a multiple tags on a single graph.

  • Click the Stacked icon to generate multiple plots stacked vertically.

Autoscaling

When an Overlaid graph is visible, the Autoscale command is available.

Right-click in the plot area of an Overlaid graph and select Autoscale from the pop-up menu:

All the values of all tags are re-scaled to range from 0 to 10. Each tags minimum value is mapped to 0 and each tags maximum value is mapped to 10. When Autoscale is active plot lines for all tags are visible.

Specifying a Graph Type

The Type command activates a submenu containing graph type commands.

  • The Trend command shows data as a trend line, plotted either in ascending time stamp or ascending sample number order.

  • The Scatter command shows data in a scatter plot format.

  • The Histogram command shows data in a histogram format.

Or, in the Toolbar of a visible Graph:

  • Click the Trend icon to show data as a trend line.

  • Click the Scatter icon to show data in a scatter plot format.

  • Click the Histogram icon to show data in a histogram format.

Using the Input Data Graph to Identify Bad Data

To identify bad data using the Input Data Graph:

  1. As described earlier, activate an area selection tool.

  2. Highlight a section of the graph by dragging the mouse. (The picture below illustrates use of the horizontal area selection tool).

  3. Right click on the highlighted area to activate a pop-up menu that can be used to mark the area good or bad, or shift to the Input Data Grid view.

    Data samples that are marked bad are displayed in red.

    Note: If you click Show in Grid, a grid view of the data will be displayed. If an area of the graph view is highlighted, the top visible row in the grid view will correspond to the beginning of the highlighted area of the graph view. If Zoom In is selected to obtain better resolution in a highlighted area, the Zoom In command changes to Zoom Out. Clicking Zoom Out will then restore the graph to the original resolution.

The Model Predictions Graph

The Model Predictions graph is accessed through the drop-down menu located to the right of the Build / Run Inferential Sensor box. Click the Drop Menu button and select Model Predictions (Graph), as illustrated in the following two figures.

Selecting Model Predictions (Graph) in Build Mode:

Selecting Model Predictions (Graph) in Run Mode:

The Model Predictions graph is a trend plot that shows the models predicted value for the dependent variable, and also the corresponding measured values, where these exist. This is one of the fundamental analysis tools for assessing the quality of your model.

You can also optionally select additional result variables to plot on this graph, such as the steady state predictions, absolute prediction error (absolute difference between measured and predicted values), and relative prediction error. This latter shows the absolute prediction error as a percentage of the corresponding measured value. These are all useful to assess the accuracy of the model.

You can also superimpose one or more of the models input variables as trend plots on this graph. This is one of the tools you can use to understand how the model maps independent measurements to an inferred value.

A model prediction graph is also actively shown during the training of a Monotonic Neural Network model type. This chart is updated regularly during the training process to show the improving fit as the training proceeds.

The Univariate Analysis Graph

The Univariate Analysis graph is accessed through the drop-down menu located to the right of the Build Inferential Sensor box. Click the Drop Menu button and select Univariate Analysis (Graph):

The Univariate Analysis graph shows how the value of the dependent variable changes with respect to the value of a single independent variable. The horizontal axis of the trend plot represents values of the selected independent variable increasing across the variable's entire range of values, with the vertical axis representing the corresponding values of inferred variable. The vertical red lines on the graph indicate the minimum and maximum values in the training dataset for the independent variable. The univariate plot extends outside the range of the training data in order to allow an assessment of the extrapolation capability of your model.

By default, complementary variables (selected independent variables) are kept constant at the mid-point of their range (their Average Value). If you do not want to use the mid-point value for the complementary variables, then:

  1. Right-click on the grid to display the context menu.

  2. Left-click Evaluations Options... to open the Evaluations Options dialog box.

The Evaluations Options dialog box contains the following options:

  • Values at Mid-points: Average values are used for all complementary variables.

  • Random Values: Random values are generated for all complementary variables.

  • Values from Data Set: Values from a row of the collected data set will be used for the complementary variables. You may select the row of the data set that is used by using the scroll bar. The text box under the scroll bar will display the selected row number.

Note:  Selections made in the Evaluations Options dialog box are immediately reflected in the grid or chart when you click the Apply button.

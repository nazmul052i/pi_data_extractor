Date/Time Property Page

The Date/Time property page is part of the Expert Parameters dialog that is accessed through the Expert button on the File Format dialog. (The File Format dialog is displayed when a data file is first loaded from the Data Specification dialog or when the Format... button on the Data Specification dialog is clicked). The Date/Time property page is used to define the format of date and time fields that occur in a data file:

Note:  To jump directly to the IQmodel Reference topic that describes a specific section or tab of the Experts Parameters dialog, click the area that corresponds to the section or tab in the illustration above.

IQmodel checks values in fields defined as dates or times to ensure that:

  • The year is greater than or equal to 1970 and less than or equal to 2038.

  • The month is valid (1 through 12) or a standard 3-character month name.

  • The day is valid (1 through 31).

  • Seconds range from 0 to 59.

  • Minutes range from 0 to 59.

  • Hours range from 0 to 23.

Note:  Date and time field locations within a larger data record are specified on the main File Format dialog.

The Date section contains drop-down lists that define the content of each sub-field in a date. A date consists of up to three sub-fields, with a separator between sub-fields.

A date component (Year, Month, Day) can be in any sub-field. Drop-down lists for each sub-field position contain all possible date component choices for the position. Select the appropriate value from the list, as illustrated below for the first sub-field:

Date sub-field drop-down list elements have the following meaning:

  • YYYY - A 4-digit year (for example 1998)

  • YY - A 2-digit year (for example 98)

  • MMMM - A full month name (for example, April)

  • MMM - A 3-character month name abbreviation (for example, Apr).

  • DD - A 2-digit day of the month (for example, 15)

  • blank - Nothing in the field position

Date separator drop-down list options are shown below:

Date separator characters contain no semantic information – they are simply placeholders that separate sub-field components of a complete date.

The Time section contains drop-down lists that define the content of each sub-field in a time. A time consists of up to three sub-fields, with a separator between sub-fields, followed by a sub-field that identifies whether times are 12 hour or 24 hour format.

A time component (Hour, Minute, Second) can be in any of the first three sub-fields. The 12 Hour/24 Hour format identification field is the last sub-field. Drop-down lists for each sub-field position contain all possible component choices for the position. Select the appropriate value from the list, as illustrated below for the first time sub-field:

Time sub-field drop-down list elements have the following meaning:

  • hh - A 2-digit hour (for example 04)

  • mm - A 2-digit minute (for example 59)

  • ss - A 2-digit minute (for example 59)

  • blank - Nothing in the sub-field position

The 12 Hour/24 Hour format identification option list is shown below:

The list elements have the following meaning:

  • AM/PM - A 1 or 2-character sub-field containing either A, AM, P, or PM follows the time

  • Blank - No format identifier follows the time; times are 24 hour format

Time separator characters contain no semantic information – they are simply placeholders that separate sub-components of a complete time.

The Delimiters section contains check boxes that specify whether (and where) date/time field delimiters occur:

Note:  Delimiter characters are defined on the Field Definition property page.

  • If the Has Delimiter between Date and Time check box is checked, a delimiter occurs between the complete date and the complete time field (for example, 04/14/98 04:59:59PM).

  • If the check box is not checked, date and time fields are contiguous (for example, 04/14/9804:59:59PM).

  • If the Has Delimiter after Date and Time check box is checked, a delimiter occurs after the combined date and time fields (for example, 04/14/98 04:59:59PM 123.45 where 123.45 represents a data value).

  • If the check box is not checked, no delimiter occurs after the combined date and time fields (for example, 04/14/98 04:59:59PM123.45 where 123.45 represents a data value).

Bad/Missing Values Property Page

The Bad/Missing Values property page is part of the Expert Parameters dialog that is accessed through the Expert button on the File Format dialog, which is displayed when a data file is first loaded through the Data Specification dialog or when the Format... button on the Data Specification dialog is clicked. The Bad/Missing Values property page is used to define character strings that represent bad and/or missing values in a data file:

The content of other fields applies to the current selection in the Attribute: drop-down list.

The Add String: edit box is used to add new strings for identifying the currently selected attribute. To add a string, simply enter it into the Add String: edit box. As you type, the characters are immediately displayed in the Values: view. A check box to enable/disable recognizing the string during parsing is automatically placed in the view.

The Values: view contains check boxes that indicate the character string values that are available to represent the attribute selected in the drop-down list. When an item is checked, any fields in a data file that contain the corresponding character string will cause the field to be marked with the attribute (i.e., either Bad or Missing).

For example, in the picture above, there are 5 character strings available to represent Missing data: a consecutive series of spaces, the string Missing, the string N/A, the string Bad, and the string -9999. However, only fields containing the string Missing, the string N/A, or the string -9999 would actually be marked Missing during input data file parsing.

Note:  Parsing is case sensitive. If there is a possibility that several case variations might exist in the data file (for example, Missing and missing), all possible forms should be in the Values: list.

To remove a string from the list, select it in the Values: view and then press the Delete or Backspace key to delete all the characters. The five IQmodel base strings shown above cannot be removed.

When you click OK to close the Expert Parameters dialog, all strings in the Values: list will be associated with the format identified in the main File Format dialog; however, only the strings that are checked will be enabled when the format is used to parse an input file.

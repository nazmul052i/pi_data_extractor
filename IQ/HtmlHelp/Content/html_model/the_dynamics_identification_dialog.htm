The Model Dynamics Dialog

The Model Dynamics dialog is used to specify parameters that govern how model dead time and dynamics relationships for the currently selected input variables are determined.

Note:  The Model Dynamics dialog (i.e., the Analyze Dynamics box) is only available if the corresponding check box was checked in the Model Defaults dialog at the beginning of the model building process.

Dead Times Section

The Dead Times section contains a summary (based on dynamic analysis of the input data) of dead time information for each independent variable, reflecting the results of the best IQmodel search and/or user-specified values. When the dialog first opens for a new model, parameters and results default to values that reflect dead times from the Align Data step.

  • The Name column indicates each independent variable’s tag name.

  • The Group column contains the variable’s dead time group name if it is a member of a group. If the column is blank, the variable is not a member of a group.

  • The Final column contains the final dead time value for the corresponding variable if either the dead time was explicitly set by the user, or the user clicked the Accept button to accept IQmodel computed dead times. If the Final column is blank, IQmodel uses the computed value in the Result column.

  • The Min and Max columns contain the current minimum and maximum dead time limits for the IQmodel dead time search.

  • The Base column indicates the time resolution of the search method (i.e., dead times determined by IQmodel will be multiples of the Base time).

  • The Result column contains the value of the corresponding variable’s dead time after the IQmodel search has been performed.

On the Statistics: line:

  • The Variables edit box contains the total number of independent variables.

  • The "Free" Variables edit box contains the number of free, or effective variables (i.e., the number of variables that the GA will use for determining dead times).

  • The Groups edit box contains the number of groups currently defined.

The Model Status: edit box indicates the current model status at this step (Invalid, Ready, or Out of Date, as defined previously).

The Base Time: edit box in the Variables section indicates the base sampling interval for the model.

Dynamics Section

The Dynamics section contains a summary (based on dynamic analysis of the input data) of dynamic filter information for each independent variable, reflecting the results of the best genetic algorithm run and/or user-specified values.

  • The Name column indicates each independent variable’s tag name.

  • The Group column contains the variable’s group name if it is a member of a group (if the column is blank, the variable is not a member of a group).

  • The Final column contains the final dead time value for the corresponding variable if either the dead time was explicitly set by the user, or the user clicked the Accept button to accept IQmodel computed dead times. If the Final column is blank, IQmodel uses the computed value in the Result column.

  • The Min and Max columns contain the current minimum and maximum filter limits for IQmodel’s dynamic filter search.

  • The Base column indicates the time resolution of the search method (i.e., filters determined by IQmodel will be multiples of the Base time).

  • The Result column contains the value of the corresponding variable’s dynamic filter after the IQmodel search has been performed.

On the Statistics: line:

  • The Variables edit box contains the total number of independent variables;

  • The "Free" Variables edit box contains the number of free, or effective variables (i.e., the number of variables that the GA will use for determining dynamics);

  • The Groups edit box contains the number of dynamics groups defined.

The Dead Times Property Page

The Dead Times property page, shown below, allows you to change parameters that govern how dead times are computed during the Identify Model Dynamics step:

The Variables section on the Dead Times tab lists the name(s) highlighted when you clicked Properties... on the pop-up menu.

  • The Name column indicates the variables whose properties can be set using this page.

  • The Group column contains the variable’s group name if the variable is a member of a Group. If the column is blank, the variable is not a member of a group.

The Dead Times section allows you to specify values for the dead times that are shown in the main Model Dynamics dialog.

Note:  Dead Times are specified as times in the form of Hours:Minutes:Seconds (HH:MM:SS). Zero fields are not displayed. For example, :06 means 6 seconds, and :01:30 means 1 minute, 30 seconds. However, dead time values must be multiples of the Dead Time Base interval. If you use the edit box spinners to adjust dead times, the spinners automatically use increments of the Dead Time Base interval. If you enter a value directly into an edit box, the value is rounded to the closest multiple of the Dead Time Base interval.

  • If the User Set radio button is selected, the adjacent Dead Time: edit box is enabled, and you can explicitly set a dead time value for the variables named in the Variables section.

  • If the GA Search radio button is selected, the adjacent Minimum Dead Time:, Maximum Dead Time:, and Dead Time Base: edit boxes are enabled, and you can enter values for these parameters.

  • The Minimum Dead Time field specifies the lower bound for dead times computed by the genetic algorithm.

  • The Maximum Dead Time field specifies the upper bound for dead times computed by the genetic algorithm.

  • The Dead Time Base: field specifies the basic Dead Time interval. The basic Dead Time interval is, in effect, the base sampling interval of input data.

The Dynamics Property Page

The Dynamics property page, shown below, allows you to change parameters that govern how model dynamic relationships are computed:

The Variables section on the Dynamics tab lists the name(s) highlighted when you clicked Properties... on the pop-up menu.

  • The Name column indicates the variables whose properties can be set using this page.

  • The Group column contains the variable’s group name if the variable is a member of a Group. If the column is blank, the variable is not a member of a group.

The Dynamics section allows you to specify values for the filters that are shown in the main Model Dynamics dialog.

Note:  Filters are specified as times in the form of Hours:Minutes:Seconds (HH:MM:SS). Zero fields are not displayed. For example, :06 means 6 seconds, and :01:30 means 1 minute, 30 seconds. You can either enter a value directly into an edit box, or use the spinners in the edit box to increase or decrease a value.

  • If the User Set radio button is selected, the adjacent Filter: edit box is enabled, and you can explicitly set a filter value for the variables named in the Variables section.

  • If the GA Search radio button is selected, the adjacent Minimum Filter:, Maximum Filter:, and Filter Base: edit boxes are enabled, and you can enter values for these parameters.

  • The Minimum Filter field specifies the lower bound for filters computed by the genetic algorithm.

  • The Maximum Filter field specifies the upper bound for filters computed by the genetic algorithm.

  • The Filter Base: field specifies the basic Filter interval. The basic Filter interval is the input data sampling interval used during dynamic filter calculations.

The Tools Menu

The Tools menu provides access to the Options: dialog:

The Options: dialog contains a Program tab and a Modeling tab.

Note:  The Options: dialog is not available when a model is open in Run Mode, and the Modeling tab is not visible when a model is open in Run Mode.

The Program Tab

The Program tab allows you to specify:

  • Warning message display preferences.

  • Default directory paths.

Warning Messages Section

The Warning Messages section allows you to turn warning messages on and off. When a check box contains a check mark, the corresponding type of warning will be displayed.

  • The Display invalidation warnings box refers to the warning that subsequent model steps will be invalidated if you proceed with the current operation.

  • The Display inconsistency warnings box refers to the warning that is displayed when you change GA parameters but do not run the GA again.

File Paths Section

The File Paths section allows you to specify the default locations for files used by IQmodel.

  • The Data: edit box contains the location for input data files.

  • The Model: edit box contains the location for IQmodel development (*.iqm) files.

  • The Runtime: edit box contains the location for IQmodel runtime (*.iqr) files generated for use by other AspenTech Plantelligenceâ„¢ online monitoring and control products.

The Modeling Tab

The Modeling tab allows you to specify:

  • Default model components to include.

  • Default model building steps for which you want to set parameters and have IQmodel perform calculations as the structure of the inferential sensor model is defined.

Compenents Section

The Components section allows you to specify which model components you wish to build.

  • The Sensor Validation check box, when checked, indicates that a sensor validation model should be built.

  • The Inferential Sensor check box, when checked, indicates that an inferential sensor model should be built.

Note:  If the Sensor Validation check box is not checked in the Model Defaults dialog when you start to create a new model, it cannot be checked later. A sensor validation model can, however, be removed by unchecking the Sensor Validation box.

Model Identification Section

The Model Identification section allows you to specify which of the model building steps related to Inferential Sensor model identification should be performed (i.e., values should be computed) by IQmodel.

  • The Align Data check box, when checked, indicates that IQmodel should be used to align model data (compute nominal dead times) in the Align Data step, and to determine dynamic dead time relationships in the Analyze Dynamics step. If the Align Data check box is not checked, the search strategy for both nominal dead times and dynamic dead times is User Set, all dead time values are set to 0, and the Align Data step is made unavailable (the box appears dimmed). See The Data Alignment Dialog and The Model Dynamics Dialog in for additional information.

  • The Select Variables check box, when checked, indicates that IQmodel should be used to select model variables in the Select Variables step. If the Select Variables check box is not checked, the status for all input variables is Active, and the Select Variables box is made unavailable (the box appears dimmed). See The Variable Selection Dialog in IQmodel Reference for additional information.

  • The Analyze Dynamics check box, when checked, indicates that IQmodel should be used to analyze model dynamics and compute filter values in the Analyze Dynamics step. If the Analyze Dynamics check box is not checked, the search strategy for filter identification is User Set, dynamic filter values are set to 0, and the Analyze Dynamics step is made unavailable (the corresponding box appears dimmed). See The Model Dynamics Dialog in IQmodel Reference for additional information.

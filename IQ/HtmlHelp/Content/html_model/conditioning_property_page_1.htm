Processing Bad Input Data

Data samples are considered bad if they are:

  • Marked bad or missing during parsing of the input data file (using value definitions specified on the Bad/Missing Values property page);

  • Explicitly marked bad in the Input Data Grid or the Input Data Graph, or in

  • ÎŸutside limits set through the Properties: dialog for a variable during the Condition Data step.

The Replace Bad Input Samples section on the Conditioning tab contains parameters that govern how bad data samples are processed:

  • The Maximum Consecutive: edit box indicates the maximum number of consecutive bad samples that will be processed (using the technique selected in the Method: drop-down list).

    Note:  If the number of consecutive bad data samples exceeds the Maximum Consecutive value, none of the bad samples in the series will be processed and replaced. The number of bad samples not processed (i.e., the number of bad samples in the series) will be reflected in the Bad column in the Variables list view on the main Data Conditioning dialog.

  • The Method: drop-down list allows you to specify how bad data samples should be handled.

    • The Interpolate method performs a linear interpolation between the last good data value preceding a series of bad data points, and the first good data value following the same series of bad data points.

    • The Last Good Sample method replaces all the values in a series of bad data with the last good data value preceding the series.

    • The Average of Good Samples method replaces each bad data point with the average of all good data samples.

Averaging Input Data

The Time-Average Inputs section on the Conditioning tab contains parameters that govern averaging of input data samples for the tag:

  • If the Output Sample to Output Sample radio button is selected, the value of each independent variable data sample will be the average of its values during the period that begins with the first value after the previous dependent variable data sample and continues through the value of the independent variable at the current time stamp.

  • If the Using Fixed Time radio button is selected, the number of data samples to average is one sample more than the result of dividing the value in the Averaging Time: edit box by the value in the Base Time: edit box (i.e., 1 + AveragingTime/BaseTime).

Validating a Model

The Validate Model section on the Conditioning tab allows you to specify parameters that govern model validation (using samples from the training dataset):

The Method: drop-down list allows you specify how validation samples should be selected:

  • When Random Selection is the method, the fraction of data (Validation Rate) you specify is randomly distributed throughout the dataset.

  • When Contiguous Block is the method, the starting point of the block is randomly selected, then the block of data corresponding to the fraction you specify is reserved for validation.

With the Validation Rate: edit box you specify the portion (as a decimal fraction) of the dataset you want to reserve for validation.

With the Random Seed: edit box you specify a seed value for the random number generator used to select input data samples for validation.

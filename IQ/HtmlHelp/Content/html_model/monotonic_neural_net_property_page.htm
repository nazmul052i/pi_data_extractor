Monotonic Neural Network Property Page

The Monotonic Neural Network property page shown below appears in the Expert Parameters dialog for the Build Inferential Sensor step. Monotonic Neural Net parameters govern the configuration of the neural network that is used when the Monotonic Neural Network model is selected:

Note: The Monotonic Neural Network page is only visible when the Monotonic Neural Network model type is selected.

  • The Number of Hidden Nodes edit box allows you to set the number of hidden nodes that the neural network contains. This is directly related to the number of equations that the neural network represents. If you double the number of hidden nodes, you approximately double the number of equations, and, consequently, the number of model parameters that must be estimated from the data. Setting the hidden nodes represents a trade-off between the accuracy of the model on the training data and the generalization ability of the neural network. More hidden nodes allow the neural network to configure itself to more accurately fit the training data, but this may be at the expense of not fitting data that is not represented in the training set.

    Setting the number of hidden nodes is often a manual iterative process in which you use the graphical analysis tools and model statistics to determine whether to increase or decrease the number of hidden nodes. A good tool in this analysis is to label some of the data as validation data. This data is not used in training the neural network; comparable performance on both the training data and validation data is indicative of a good model even if it may not apparently be as accurate as some other models.

    Note:  If the training dataset is noisy, overfitting of the neural network to the training data can be avoided by limiting the number of hidden nodes.

  • The Weight Decay edit box allows you set another training parameter that affects generalization. This is the weight decay parameter which acts as a "forgetting" mechanism for the training. A neural network is trained by showing it the training data many times. At each pass of the data, the neural network output is compared with the measured output from the plant. If there are consistent relationships in the data, then these relationships will be consolidated at each pass of the data. Spurious or isolated relationships in the data will be filtered out by the weight decay.

    As with Number of Hidden Nodes, Weight Decay is usually set by a manual iterative process. The advantage of Weight Decay over Number of Hidden Nodes is that there is less trade-off with accuracy; you can keep the number of hidden nodes high, but still get good generalization.

  • The Maximum Number of Iterations edit box allows you to set the number of iterations used by the optimization algorithm that is adjusting the neural network parameters. Use the default the first time you train the neural network. If the status dialog displayed during training indicates that further improvements are still occurring in the last few iterations (especially improvements in the validation data result), then you should increase the Maximum Number of Iterations. If, however, over the course of several training runs, the optimum is always achieved much earlier than the final iteration, you can reduce Maximum Number of Iterations.

Running an Inferential Sensor Model

To run an inferential sensor model with data from files other than those used for training, IQmodel must be in Run Mode, and data source(s) must be specified through the Data Specification dialog.

After datasets have been loaded, click the Run Inferential Sensor box to access the Model dialog, illustrated below:

The Variables section, illustrated below, contains information about model variables:

  • The Name column contains the names of all variables used in the model.

  • The Status column indicates each variable’s status. Dep indicates that the corresponding variable is the dependent variable. Ind indicates that the corresponding variable is an independent variable.

  • The Dead Time column contains model dead time values for the corresponding variables.

  • The Filter column contains model filter values for the corresponding variables.

The Results section, illustrated below, contains results after the model is run using data from the currently loaded file:

Note:  When the Model dialog is first opened, the list view in the Results section is blank, and the Run button is disabled.

When you click the Run button, the model is executed using data from the currently loaded file.

Note:  If the file does not contain sufficient data for each of the independent variables, an error message is displayed and the model is not run.

If the file contains data for all independent variables, when the model finishes the top row of the list view will contain statistics computed for the dependent variable, based on the model. If the data file also contained data for the dependent variable, statistics in the second row reflect computations based on file data for the dependent variable.

  • The Samples column contains the number of data samples used to compute statistics values displayed in the other columns.

  • The Variance column indicates the variance of the dependent variable.

  • The RMS Error column indicates the root mean squared error of the dependent variable.

  • The Avg. Error column indicates the average error of the dependent variable.

  • The Max. Error column indicates the maximum error of the dependent variable.

  • The Correlation column indicates the correlation of the dependent variable.

  • The Model: box indicates the model type.

  • The Latent Variables: box indicates the number of latent variables in the model.

    Note:  The value is 0 for Monotonic Neural Network models.

  • The Hidden Units: box contains the number of hidden units in the neural network component of the model if the model type is Hybrid: PLS – Neural Network or Monotonic Neural Network. In the former case, the number of hidden nodes is selected by the neural net training algorithm, so this text box provides the results of this automatic selection.

    Note:  The value is 0 for Linear and Fuzzy PLS models.

  • The Model Status: box indicates the current model status at this step (Invalid, Ready, or Out of Date, as previously defined).

Flatline Period

The Flatline Period is the length of time, in absolute time units, that a variable’s values must remain below the Flatline Threshold to cause a Flatline fault (IQmodel ensures that the period is a multiple of the model’s Base Time). In other words, when a data sample for the variable falls below the Flatline Threshold, a timer for the Flatline Period is started, but no fault is generated (assuming Flatline detection is enabled). If all data samples subsequently collected during the Flatline Period remain below the Flatline Threshold, they are also considered good; however, if the first sample collected after the Flatline Period has elapsed is below the Flatline Threshold, the sample is marked bad and a Flatline fault is generated. Conversely, if before the Flatline Period has elapsed a data sample equals or exceeds the Flatline Threshold, the Flatline Period timer is turned off and no fault is generated.
